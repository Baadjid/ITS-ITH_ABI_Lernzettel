📊 Algorithmen & Datenstrukturen

Die Grundlagen effizienter Programmierung verstehen und anwenden

🎯 Was du in diesem Kapitel lernst

Algorithmen und Datenstrukturen sind das Herzstück der Informatik. Du lernst, wie Computer Probleme systematisch lösen und wie Daten effizient organisiert werden können. Nach diesem Kapitel weißt du, warum manche Programme schneller laufen als andere und wie du selbst effiziente Lösungen programmieren kannst.

🧮 1. Was sind Algorithmen?

Definition: Ein Algorithmus ist eine Verarbeitungsvorschrift, die aus einer endlichen Folge von eindeutig ausführbaren Anweisungen besteht, mit der man eine Vielzahl gleichartiger Aufgaben lösen kann.

📋 Die Eigenschaften eines Algorithmus

1. Eindeutigkeit

Jeder Schritt des Algorithmus ist klar definiert. Es gibt keine Mehrdeutigkeiten oder Interpretationsspielräume.

Beispiel: "Addiere 5 zu einer Zahl" ist eindeutig. "Addiere etwas zu einer Zahl" wäre nicht eindeutig.

2. Endlichkeit (Finitheit)

Ein Algorithmus besteht aus endlich vielen Anweisungen und muss nach endlich vielen Schritten terminieren.

3. Ausführbarkeit (Effektivität)

Jede Anweisung muss für den Ausführenden (Computer oder Mensch) verständlich und ausführbar sein.

4. Determinismus

Bei gleichen Eingaben liefert der Algorithmus immer die gleichen Ergebnisse.

🏗️ Grundbausteine von Algorithmen

1
Sequenz: Anweisungen werden nacheinander abgearbeitet
int a = 5; int b = 10; int summe = a + b; // Wird nach den oberen Zeilen ausgeführt
2
Verzweigung: Abhängig von Bedingungen werden unterschiedliche Wege genommen
if (alter >= 18) { System.out.println("Du darfst wählen"); } else { System.out.println("Du bist noch zu jung"); }
3
Schleife: Anweisungen werden wiederholt ausgeführt
for (int i = 0; i < 5; i++) { System.out.println("Durchlauf: " + i); }

📦 2. Was sind Datenstrukturen?

Definition: Datenstrukturen sind Objekte zur Speicherung und Organisation von Daten. Sie bestimmen, wie Daten angeordnet, verknüpft und verwaltet werden, um effizienten Zugriff zu ermöglichen.

📊 Statische vs. Dynamische Datenstrukturen

📏 Statische Datenstrukturen

  • Haben eine festgelegte Größe
  • Größe wird bei der Erstellung definiert
  • Schneller Zugriff durch feste Speicherplätze
  • Beispiel: Arrays

📈 Dynamische Datenstrukturen

  • Größe kann sich zur Laufzeit ändern
  • Flexibler, aber oft etwas langsamer
  • Speicher wird nach Bedarf allokiert
  • Beispiele: Listen, Stacks, Queues

📋 3. Arrays (Felder) - Die Grundlage

Array: Ein Array ist eine statische Datenstruktur, die mehrere Elemente gleichen Datentyps in einer festen Reihenfolge speichert. Jedes Element hat einen Index (Position).

🔄 4. Dynamische Datenstrukturen

📚 Stapel (Stack) - LIFO-Prinzip

Stack (Stapel): Funktioniert nach dem LIFO-Prinzip (Last In, First Out). Wie ein Stapel Teller - das zuletzt draufgelegte Element wird als erstes wieder weggenommen.

Stack-Visualisierung:

Element 3 (Top)
Element 2
Element 1 (Bottom)

Push: Neues Element oben drauflegen
Pop: Oberstes Element entfernen und zurückgeben

🔧 Stack-Operationen

1
push(element): Fügt ein Element oben auf den Stapel hinzu
2
pop(): Entfernt das oberste Element und gibt es zurück
3
top(): Zeigt das oberste Element an, ohne es zu entfernen
4
isEmpty(): Prüft, ob der Stack leer ist

🚌 Warteschlange (Queue) - FIFO-Prinzip

Queue (Schlange): Funktioniert nach dem FIFO-Prinzip (First In, First Out). Wie eine Warteschlange - wer zuerst kommt, wird zuerst bedient.

Queue-Visualisierung:

Element 1
Element 2
Element 3

Enqueue: Element hinten anstellen
Dequeue: Vorderstes Element entfernen

🔧 Queue-Operationen

1
enqueue(element): Fügt ein Element am Ende der Schlange hinzu
2
dequeue(): Entfernt das vorderste Element und gibt es zurück
3
head(): Zeigt das vorderste Element an
4
isEmpty(): Prüft, ob die Queue leer ist

📝 Dynamische Liste (ArrayList)

Dynamische Liste: Eine veränderbare Sammlung von Elementen, auf die über Indizes zugegriffen werden kann. Im Gegensatz zu Arrays kann die Größe zur Laufzeit verändert werden.
import java.util.ArrayList; // ArrayList erstellen ArrayList<Integer> liste = new ArrayList<>(); // Elemente hinzufügen liste.add(10); // [10] liste.add(20); // [10, 20] liste.add(1, 15); // [10, 15, 20] - an Index 1 einfügen // Elemente abrufen int element = liste.get(0); // 10 // Element entfernen liste.remove(1); // [10, 20] - Index 1 entfernen // Größe prüfen int groesse = liste.size(); // 2

🌳 5. Binärbäume - Hierarchische Datenstrukturen

Binärbaum: Eine hierarchische Datenstruktur, bei der jeder Knoten maximal zwei Kindknoten hat (linkes und rechtes Kind). Der oberste Knoten heißt Wurzel.

🔍 Binärbaum-Terminologie

  • Wurzel (Root): Der oberste Knoten
  • Blatt (Leaf): Knoten ohne Kinder
  • Innerer Knoten: Knoten mit mindestens einem Kind
  • Tiefe/Level: Abstand zur Wurzel

Beispiel eines Binärbaums:

8
3
10
1
6
14

Wurzel: 8 | Blätter: 1, 6, 14 | Tiefe: 3 Ebenen

🔄 Baum-Traversierung (Durchlaufen)

Es gibt drei wichtige Arten, einen Binärbaum systematisch zu durchlaufen:

📍 Preorder

Reihenfolge: Wurzel → Links → Rechts

Beispiel: 8, 3, 1, 6, 10, 14

void preorder(Knoten k) { if (k != null) { System.out.print(k.wert); preorder(k.links); preorder(k.rechts); } }

📍 Inorder

Reihenfolge: Links → Wurzel → Rechts

Beispiel: 1, 3, 6, 8, 10, 14

void inorder(Knoten k) { if (k != null) { inorder(k.links); System.out.print(k.wert); inorder(k.rechts); } }

📍 Postorder

Reihenfolge: Links → Rechts → Wurzel

Beispiel: 1, 6, 3, 14, 10, 8

void postorder(Knoten k) { if (k != null) { postorder(k.links); postorder(k.rechts); System.out.print(k.wert); } }
💡 Merkhilfe:
• Preorder: Wurzel kommt vor den Kindern
• Inorder: Wurzel kommt zwischen den Kindern
• Postorder: Wurzel kommt nach den Kindern

🔍 Binärer Suchbaum

Binärer Suchbaum: Ein spezieller Binärbaum, bei dem für jeden Knoten gilt: Alle Werte im linken Teilbaum sind kleiner, alle Werte im rechten Teilbaum sind größer als der Knotenwert.

🔧 Operationen im binären Suchbaum

1
Suchen nach Wert x:
  • Wenn aktueller Knoten = x → Gefunden!
  • Wenn aktueller Knoten < x → Rechts weitersuchen
  • Wenn aktueller Knoten > x → Links weitersuchen
  • Wenn leerer Teilbaum → Nicht gefunden
2
Einfügen von Wert x:
  • Suche die richtige Position (wie beim Suchen)
  • Füge x an der leeren Stelle ein
3
Löschen von Wert x:
  • Fall 1: Knoten ist Blatt → Einfach entfernen
  • Fall 2: Knoten hat ein Kind → Durch Kind ersetzen
  • Fall 3: Knoten hat zwei Kinder → Durch Vorgänger oder Nachfolger ersetzen

🔀 6. Sortieralgorithmen

Sortieralgorithmus: Ein Verfahren, um Daten in eine bestimmte Reihenfolge zu bringen (meist aufsteigend oder absteigend). Die Effizienz verschiedener Algorithmen unterscheidet sich erheblich.

📊 Algorithmus-Komplexität verstehen

Was bedeutet "Komplexität"?
Die Komplexität beschreibt, wie sich die Laufzeit eines Algorithmus verändert, wenn die Datenmenge größer wird. Stell dir vor, du sortierst erst 10 Zahlen, dann 100, dann 1000...
Komplexität Name Bei 10 Elementen Bei 100 Elementen Bei 1000 Elementen
O(n) Linear 10 Schritte 100 Schritte 1.000 Schritte
O(n log n) Linearithmisch 33 Schritte 664 Schritte 9.966 Schritte
O(n²) Quadratisch 100 Schritte 10.000 Schritte 1.000.000 Schritte
⚠️ Wichtig: Bei 1.000.000 Elementen benötigt ein O(n²) Algorithmus im schlechtesten Fall 1.000.000.000.000 Operationen, während O(n log n) nur etwa 20.000.000 benötigt!

🫧 Bubblesort - Der einfachste Sortieralgorithmus

Bubblesort: Vergleicht benachbarte Elemente paarweise und vertauscht sie, wenn sie in falscher Reihenfolge stehen. Wie Luftblasen steigen die größten Elemente nach oben.
// Bubblesort Implementation public static void bubblesort(int[] array) { int n = array.length; // Äußere Schleife: n-1 Durchläufe for (int i = 0; i < n - 1; i++) { // Innere Schleife: Paarweise Vergleiche for (int j = 0; j < n - i - 1; j++) { // Wenn linkes Element größer als rechtes if (array[j] > array[j + 1]) { // Vertauschen int temp = array[j]; array[j] = array[j + 1]; array[j + 1] = temp; } } } }

⚡ Quicksort - Der schnelle Algorithmus

Quicksort: Ein effizienter Divide-and-Conquer-Algorithmus, der ein Pivot-Element auswählt und das Array in kleinere und größere Elemente unterteilt.
// Quicksort Implementation public static void quicksort(int[] array, int low, int high) { if (low < high) { // Partitioniere das Array int pi = partition(array, low, high); // Sortiere Elemente vor und nach der Partition quicksort(array, low, pi - 1); quicksort(array, pi + 1, high); } } private static int partition(int[] array, int low, int high) { int pivot = array[high]; // Letztes Element als Pivot int i = (low - 1); for (int j = low; j < high; j++) { if (array[j] < pivot) { i++; // Tausche array[i] und array[j] int temp = array[i]; array[i] = array[j]; array[j] = temp; } } // Tausche array[i+1] und array[high] (Pivot) int temp = array[i + 1]; array[i + 1] = array[high]; array[high] = temp; return i + 1; }

📈 Vergleich der Sortieralgorithmen

Algorithmus Best Case Average Case Worst Case Speicher Stabil?
Bubblesort O(n) O(n²) O(n²) O(1) Ja
Quicksort O(n log n) O(n log n) O(n²) O(log n) Nein
Mergesort O(n log n) O(n log n) O(n log n) O(n) Ja

🔍 7. Suchalgorithmen

Suchalgorithmus: Ein Verfahren, um ein bestimmtes Element in einer Datenstruktur zu finden. Die Effizienz hängt stark davon ab, ob die Daten sortiert sind oder nicht.

📍 Lineare Suche (Sequenzielle Suche)

Lineare Suche: Durchsucht die Daten Element für Element von Anfang bis Ende, bis das gesuchte Element gefunden wird oder das Ende erreicht ist.
// Lineare Suche Implementation public static int lineareSuche(int[] array, int gesucht) { for (int i = 0; i < array.length; i++) { if (array[i] == gesucht) { return i; // Index des gefundenen Elements } } return -1; // Element nicht gefunden }

🎯 Binäre Suche

Binäre Suche: Funktioniert nur bei sortierten Daten. Teilt den Suchbereich in jeder Iteration halbiert, indem das mittlere Element mit dem gesuchten verglichen wird.
// Binäre Suche Implementation public static int binaereSuche(int[] array, int gesucht) { int links = 0; int rechts = array.length - 1; while (links <= rechts) { int mitte = (links + rechts) / 2; if (array[mitte] == gesucht) { return mitte; // Gefunden } else if (array[mitte] < gesucht) { links = mitte + 1; // Rechte Hälfte } else { rechts = mitte - 1; // Linke Hälfte } } return -1; // Nicht gefunden }

📊 Vergleich der Suchalgorithmen

Algorithmus Voraussetzung Best Case Average Case Worst Case
Lineare Suche Keine O(1) O(n) O(n)
Binäre Suche Sortiert O(1) O(log n) O(log n)

🔄 8. Rekursion vs. Iteration

Rekursion: Eine Funktion ruft sich selbst auf, um ein Problem zu lösen, indem es in kleinere Teilprobleme zerlegt wird.
Iteration: Wiederholung von Anweisungen durch Schleifen (for, while).

🧮 Beispiel: Fakultät berechnen

Fakultät: n! = n × (n-1) × (n-2) × ... × 2 × 1
Beispiel: 5! = 5 × 4 × 3 × 2 × 1 = 120

🔄 Rekursive Lösung

public static int fakultaetRekursiv(int n) { // Basisfall if (n <= 1) { return 1; } // Rekursiver Aufruf return n * fakultaetRekursiv(n - 1); }

Denkweise: 5! = 5 × 4!, 4! = 4 × 3!, ...

🔁 Iterative Lösung

public static int fakultaetIterativ(int n) { int ergebnis = 1; for (int i = 1; i <= n; i++) { ergebnis = ergebnis * i; } return ergebnis; }

Denkweise: 1×1×2×3×4×5 = 120

📊 Vor- und Nachteile

Aspekt Rekursion Iteration
Verständlichkeit Oft natürlicher und eleganter Manchmal komplexer zu verstehen
Speicherverbrauch Höher (Call Stack) Niedriger
Geschwindigkeit Langsamer (Funktionsaufrufe) Schneller
Codekürze Meist kürzer Oft länger

📚 11. Zusammenfassung & Klausurvorbereitung

🎯 Die wichtigsten Konzepte

📊 Datenstrukturen

  • Arrays: Feste Größe, schneller Zugriff
  • Listen: Dynamische Größe, flexibel
  • Stacks: LIFO-Prinzip
  • Queues: FIFO-Prinzip
  • Bäume: Hierarchische Struktur

🔀 Sortieralgorithmen

  • Bubblesort: O(n²), einfach
  • Quicksort: O(n log n), schnell
  • Mergesort: O(n log n), stabil
  • Insertionsort: Gut für kleine Arrays

🔍 Suchalgorithmen

  • Linear: O(n), unsortierte Daten
  • Binär: O(log n), sortierte Daten
  • Baum-Suche: O(log n) im Durchschnitt

💡 Tipps für die Klausur

📋 Checkliste

  • ✅ Verstehe die Komplexitäten (O-Notation)
  • ✅ Kenne die Vor-/Nachteile jedes Algorithmus
  • ✅ Übe das manuelle Durchführen von Sortier-/Suchalgorithmen
  • ✅ Verstehe Rekursion und deren Grenzen
  • ✅ Wisse, wann welche Datenstruktur sinnvoll ist

🧠 Häufige Klausurfragen

  1. Sortiere Array X mit Algorithmus Y (Schritt für Schritt)
  2. Vergleiche Laufzeiten verschiedener Algorithmen
  3. Implementiere eine rekursive Funktion
  4. Wähle die beste Datenstruktur für Szenario Z
  5. Analysiere die Komplexität eines gegebenen Codes
