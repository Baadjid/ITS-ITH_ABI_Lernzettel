🎯 Was ist ein Betriebssystem?

Die Grundlagen verstehen - von Null auf Hundert

🤔 Stell dir vor...

Du willst ein Foto bearbeiten, Musik hören und gleichzeitig im Internet surfen. Wer sorgt dafür, dass dein Computer all diese Aufgaben gleichzeitig bewältigen kann, ohne dass sich die Programme gegenseitig stören? Das ist die Aufgabe des Betriebssystems!

📖 Definition

Ein Betriebssystem (OS = Operating System) ist eine Sammlung von Programmen, die als Vermittler zwischen Hardware und Anwendungssoftware fungiert. Es verwaltet alle Ressourcen des Computers und stellt anderen Programmen Dienste zur Verfügung.

🎭 Die Rolle des Betriebssystems

Das OS ist wie der Manager eines Hotels: Es verwaltet die Zimmer (Speicher), koordiniert das Personal (Prozesse), sorgt für Sicherheit (Zugriffsrechte) und stellt sicher, dass alle Gäste (Programme) zufrieden sind.

🌟 Bekannte Beispiele

  • Windows (Microsoft)
  • macOS (Apple)
  • Linux (Open Source)
  • Android (Mobile)
  • iOS (Mobile)

🏗️ Schichten-Architektur eines Betriebssystems

👤 Benutzer
📱 Anwendungsprogramme (Word, Browser, Spiele)
⚙️ Systemdienste & API
🔧 Betriebssystem-Kern (Kernel)
🖥️ Hardware (CPU, RAM, Festplatte)

Jede Schicht kommuniziert nur mit der direkt darüber- und darunterliegenden Schicht

⚙️ Aufgaben und Funktionen

Was macht ein Betriebssystem eigentlich den ganzen Tag?

🎯 Prozessmanagement

Verwaltung aller laufenden Programme (Prozesse). Das OS entscheidet, welches Programm wann die CPU verwenden darf und sorgt dafür, dass Programme nicht miteinander kollidieren.

Beispiel

Du hörst Musik, während du gleichzeitig einen Text schreibst. Das OS wechselt blitzschnell zwischen beiden Programmen hin und her.

🧠 Speicherverwaltung

Zuteilung und Verwaltung des Arbeitsspeichers (RAM). Jedes Programm bekommt seinen eigenen Speicherbereich, und das OS sorgt dafür, dass Programme nicht in fremde Bereiche zugreifen.

Wichtig!

Ohne Speicherverwaltung würden Programme sich gegenseitig überschreiben und zum Absturz bringen.

📁 Dateisystem-Verwaltung

Organisation und Verwaltung von Dateien und Ordnern auf Speichermedien. Das OS weiß, wo jede Datei gespeichert ist und wie man sie wiederfindet.

🔌 Geräte-Verwaltung

Kommunikation mit Hardware-Komponenten über Gerätetreiber. Drucker, Maus, Tastatur - alles wird vom OS koordiniert.

🛡️ Sicherheit & Schutz

Schutz vor unbefugtem Zugriff und Isolierung von Programmen. Das OS kontrolliert, wer was darf.

👥 Benutzer-Interface

Bereitstellung einer Schnittstelle zwischen Mensch und Maschine (GUI oder Kommandozeile).

🏗️ Architektur und Aufbau

Wie ist ein Betriebssystem intern strukturiert?

🔧 Der Kernel (Betriebssystem-Kern)

Das Herzstück des Betriebssystems. Der Kernel läuft im privilegierten Modus und hat direkten Zugriff auf die Hardware. Er ist verantwortlich für:

  • Prozess-Scheduling
  • Speicher-Management
  • Hardware-Abstraction
  • System-Calls

🏠 Betriebsarten

User Mode vs. Kernel Mode

  • User Mode: Eingeschränkte Rechte, normale Programme laufen hier
  • Kernel Mode: Vollzugriff auf Hardware, nur OS-Kernel läuft hier

Warum diese Trennung?

Sicherheit! Ein fehlerhaftes Programm im User Mode kann das System nicht zum Absturz bringen.

🔄 System Call Mechanismus

// Wenn ein Programm eine Datei öffnen möchte: 1. Programm ruft open("datei.txt") auf 2. System Call wird ausgelöst 3. Wechsel vom User Mode zum Kernel Mode 4. Kernel öffnet die Datei 5. Rückgabe an das Programm 6. Wechsel zurück zum User Mode

System Calls sind die "Hotline" zwischen Anwendungen und dem Betriebssystem

🔄 Prozesse und Threads

Wie Programme im System leben und arbeiten

🤔 Was ist der Unterschied zwischen Programm und Prozess?

Programm: Eine Datei auf der Festplatte (wie ein Kochrezept)
Prozess: Ein ausgeführtes Programm im Speicher (wie das tatsächliche Kochen)

⚡ Prozess

Ein Prozess ist eine Instanz eines ausgeführten Programms. Jeder Prozess hat:

  • Eigenen Speicherbereich
  • Eindeutige Prozess-ID (PID)
  • Eigene Ressourcen
  • Mindestens einen Thread

🧵 Thread

Ein Thread ist ein Ausführungsstrang innerhalb eines Prozesses. Mehrere Threads teilen sich:

  • Denselben Speicherbereich
  • Dieselben Ressourcen
  • Aber eigene CPU-Register
  • Eigenen Stack

🔄 Prozess-Zustände

Bereit
Ready
Laufend
Running
Wartend
Waiting
Beendet
Terminated

Bereit: Prozess kann ausgeführt werden, wartet auf CPU

Laufend: Prozess wird gerade von der CPU ausgeführt

Wartend: Prozess wartet auf ein Ereignis (z.B. Eingabe)

Beendet: Prozess ist fertig und wird aufgeräumt

Praktisches Beispiel: Webbrowser

Ein moderner Webbrowser wie Chrome erstellt für jedes Tab einen eigenen Prozess. Wenn ein Tab abstürzt, bleiben die anderen Tabs funktionsfähig. Innerhalb jedes Tab-Prozesses laufen mehrere Threads für verschiedene Aufgaben (Rendering, JavaScript, Netzwerk).

Mehrprozessbetrieb (Multitasking)

Moderne Betriebssysteme können scheinbar mehrere Programme gleichzeitig ausführen. In Wahrheit wechselt die CPU sehr schnell zwischen den Prozessen hin und her (Time-Sharing). Bei Multi-Core-Prozessoren können tatsächlich mehrere Prozesse parallel laufen.

⏰ Scheduling - Wer darf wann ran?

Wie das Betriebssystem entscheidet, welcher Prozess die CPU bekommt

🎭 Der CPU-Scheduler ist wie ein Theaterdirektor

Stell dir vor, die CPU ist eine Bühne und die Prozesse sind Schauspieler. Der Scheduler entscheidet, wer wann auf die Bühne darf und wie lange er dort bleiben kann.

🛠️ Scheduling-Algorithmen

Algorithmus Funktionsweise Vorteile Nachteile
FIFO/FCFS
First In, First Out		 Prozesse werden in der Reihenfolge abgearbeitet, wie sie ankommen Einfach, fair Lange Wartezeiten möglich
SJF
Shortest Job First		 Kürzeste Prozesse zuerst Minimiert Wartezeit Lange Prozesse können "verhungern"
Round Robin Jeder Prozess bekommt eine feste Zeitscheibe Fair, responsiv Overhead durch Kontextwechsel
Priority Scheduling Prozesse mit höchster Priorität zuerst Wichtige Aufgaben bevorzugt Prioritätsinversion möglich

Round Robin Beispiel

Zeitscheibe: 100ms Prozesse: A (300ms), B (200ms), C (100ms) Ausführungsreihenfolge: A(100ms) → B(100ms) → C(100ms fertig) → A(100ms) → B(100ms fertig) → A(100ms fertig)

Jeder Prozess bekommt 100ms CPU-Zeit, dann wird gewechselt. Das sorgt für eine faire Verteilung und responsive Systeme.

Kontextwechsel (Context Switch)

Wenn die CPU von einem Prozess zum anderen wechselt, muss der aktuelle Zustand (Register, Stack-Pointer, etc.) gespeichert und der neue Zustand geladen werden. Das kostet Zeit, aber ermöglicht Multitasking!

🔄 Verklemmung (Deadlock)

Ein gefährliches Problem: Zwei oder mehr Prozesse warten ewig aufeinander.

Beispiel Deadlock: Prozess A hat Ressource 1, braucht Ressource 2 Prozess B hat Ressource 2, braucht Ressource 1 → Beide warten ewig!

Lösungen: Deadlock-Vermeidung, -Erkennung oder -Auflösung durch das OS

🧠 Speicherverwaltung

Wie das OS den wertvollen Arbeitsspeicher verwaltet

🏢 Der Arbeitsspeicher ist wie ein Bürogebäude

Jedes Programm braucht "Büroräume" (Speicherplatz). Das OS ist der Hausmeister, der entscheidet, wer welche Räume bekommt und darauf achtet, dass niemand in fremde Büros eindringt.

🏗️ Speicher-Layout eines Prozesses

Stack (lokale Variablen, Funktionsaufrufe)
Freier Speicher
Heap (dynamische Speicherallokation)
Daten-Segment (globale/statische Variablen)
Code-Segment (Programmcode)

📊 Speicherarten

  • Stack: Lokale Variablen, automatische Verwaltung
  • Heap: Dynamisch allokierter Speicher (malloc/new)
  • Data: Globale und statische Variablen
  • Code: Der eigentliche Programmcode

🔒 Speicherschutz

Jeder Prozess hat seinen eigenen virtuellen Adressraum. Das OS sorgt dafür, dass Prozesse nicht in fremden Speicher zugreifen können.

Warum wichtig?

Ohne Speicherschutz könnte ein fehlerhaftes Programm andere Programme oder sogar das OS zum Absturz bringen!

💾 Virtueller Speicher

Jeder Prozess "denkt", er hat den ganzen Speicher für sich allein. Das OS übersetzt virtuelle in physische Adressen.

Praktischer Nutzen

Programme können größer sein als der verfügbare RAM. Nicht verwendete Teile werden auf die Festplatte ausgelagert (Swapping).

📄 Paging

Der Speicher wird in gleich große Blöcke (Pages/Seiten) aufgeteilt. Das ermöglicht effiziente Verwaltung und Fragmentierung wird vermieden.

Page-Größe: meist 4KB Virtuelle Adresse: 0x12345678 Page-Nummer: 0x12345 (obere Bits) Offset: 0x678 (untere Bits)

🔄 Swapping

Wenn der RAM voll ist, werden inaktive Pages auf die Festplatte ausgelagert. Bei Bedarf werden sie zurückgeholt.

Performance-Tipp

Zu viel Swapping (Thrashing) macht das System langsam. Mehr RAM ist oft die beste Lösung!

🧩 Fragmentierung

Interne Frag.: Verschwendung innerhalb zugeteilter Blöcke
Externe Frag.: Freie Speicherblöcke sind zu klein

Lösung: Paging eliminiert externe Fragmentierung!

📁 Dateisysteme

Wie das OS Dateien und Ordner verwaltet

📚 Das Dateisystem ist wie eine riesige Bibliothek

Millionen von "Büchern" (Dateien) müssen so organisiert werden, dass man sie schnell findet und nichts verloren geht. Das Dateisystem ist das Ordnungssystem dieser Bibliothek.

🗂️ Aufgaben des Dateisystems

  • Speicherung: Dateien auf Speichermedien ablegen
  • Organisation: Verzeichnisstrukturen verwalten
  • Zugriff: Schnelles Finden und Öffnen von Dateien
  • Sicherheit: Zugriffsrechte kontrollieren
  • Integrität: Datenverlust verhindern

🏗️ Dateisystem-Strukturen

Hierarchische Struktur: Dateien sind in Verzeichnissen (Ordnern) organisiert, die wiederum Unterverzeichnisse enthalten können.

/ ├── home/ │ └── benutzer/ │ ├── dokumente/ │ └── bilder/ ├── system/ └── programme/

💾 Wichtige Dateisystem-Typen

Dateisystem Betriebssystem Max. Dateigröße Besonderheiten
FAT32 Windows, Universal 4 GB Einfach, weit verbreitet
NTFS Windows 16 TB Berechtigungen, Komprimierung
EXT4 Linux 16 TB Journaling, stabil
APFS macOS 8 EB Verschlüsselung, Snapshots

📋 File Allocation Table (FAT)

Eine Tabelle, die festhält, wo auf der Festplatte die einzelnen Teile einer Datei gespeichert sind.

Datei "foto.jpg" belegt Cluster: FAT[100] = 101 // Cluster 100 → 101 FAT[101] = 102 // Cluster 101 → 102 FAT[102] = EOF // Ende der Datei

🔧 Partitionen

Eine Festplatte kann in mehrere logische Bereiche aufgeteilt werden. Jede Partition kann ein eigenes Dateisystem haben.

Typische Aufteilung

C: System (Windows)
D: Programme
E: Daten

🗑️ Fragmentierung

Dateien werden nicht immer zusammenhängend gespeichert, sondern in Fragmenten über die Festplatte verteilt. Das verlangsamt den Zugriff.

Lösung

Defragmentierung ordnet die Dateien neu und legt sie zusammenhängend ab. Bei SSDs ist das nicht nötig!

📝 Journaling

Moderne Dateisysteme führen ein "Tagebuch" aller Änderungen. Bei einem Systemabsturz können unvollständige Operationen rückgängig gemacht werden.

Ablauf

1. Schreibe in Journal: "Werde Datei X ändern"
2. Ändere Datei X
3. Lösche Journal-Eintrag

🛡️ Sicherheit & Zugriffsrechte

Wie das OS für Sicherheit und kontrollierte Zugriffe sorgt

🏰 Sicherheit ist wie ein Burgensystem

Das Betriebssystem ist die Burg, die alle Ressourcen schützt. Es gibt verschiedene Sicherheitsebenen - von der äußeren Mauer (Login) bis zu den innersten Kammern (Kernel-Speicher).

👤 Benutzerverwaltung

Das OS unterscheidet zwischen verschiedenen Benutzern und Benutzergruppen mit unterschiedlichen Rechten.

  • Administrator/Root: Vollzugriff auf das System
  • Normale Benutzer: Eingeschränkte Rechte
  • Gäste: Minimale Rechte
  • Systemdienste: Spezielle Service-Accounts

📋 Zugriffsrechte

Jede Datei und jedes Verzeichnis hat Zugriffsrechte, die bestimmen, wer was darf.

Unix/Linux Rechte: rwxrwxrwx ├┼┼─ Owner (Besitzer) ├┼─ Group (Gruppe) └─ Other (Alle anderen) r = read (lesen) w = write (schreiben) x = execute (ausführen)

🔐 Authentifizierung

Wie das System überprüft, dass Benutzer wirklich die sind, für die sie sich ausgeben.

  • Passwörter: "Etwas, das du weißt"
  • Biometrie: "Etwas, das du bist"
  • Token/Cards: "Etwas, das du hast"
  • 2FA: Kombination mehrerer Methoden

🛡️ Sicherheitsmechanismen

Verschiedene Schutzmaßnahmen des Betriebssystems:

  • Memory Protection: Prozesse können nicht in fremden Speicher
  • Sandbox: Programme laufen in isolierten Umgebungen
  • Access Control Lists: Detaillierte Rechtevergabe
  • Antivirus-Integration: Schutz vor Schadsoftware

Prinzip der minimalen Rechte

Jeder Benutzer und jedes Programm sollte nur die Rechte haben, die für seine Aufgabe unbedingt nötig sind. Das minimiert Sicherheitsrisiken!

Windows UAC (User Account Control)

Auch Administratoren arbeiten normalerweise mit eingeschränkten Rechten. Nur bei administrativen Aufgaben wird nachgefragt und temporär erhöhte Rechte vergeben.
