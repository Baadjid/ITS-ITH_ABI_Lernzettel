🎯 Was sind Dateisysteme?

Stell dir vor, du hast einen riesigen Schrank und möchtest darin tausende von Dokumenten ordentlich aufbewahren. Du brauchst ein System, um zu wissen, wo welches Dokument liegt, wie du schnell darauf zugreifen kannst und wie du sicherstellst, dass nichts verloren geht.

Genau das macht ein Dateisystem für deinen Computer!

💡 Definition: Dateisystem

Ein Dateisystem ist die Art und Weise, wie ein Betriebssystem Dateien auf einem Speichermedium (wie einer Festplatte) organisiert, speichert und verwaltet. Es bestimmt:

  • Wie Dateien benannt werden
  • Wo sie gespeichert werden
  • Wie auf sie zugegriffen wird
  • Welche Sicherheits- und Zugriffsrechte gelten

🔍 Warum brauchen wir Dateisysteme?

Ohne Dateisystem wäre eine Festplatte nur ein riesiger Berg von Nullen und Einsen ohne jede Struktur. Das Dateisystem sorgt dafür, dass:

  • Ordnung herrscht: Dateien haben Namen und befinden sich in Ordnern
  • Schneller Zugriff möglich ist: Das System weiß, wo jede Datei liegt
  • Sicherheit gewährleistet ist: Zugriffsrechte schützen Dateien
  • Effizienz erreicht wird: Speicherplatz wird optimal genutzt

💽 Massenspeicher Festplatte

Bevor wir Dateisysteme verstehen, müssen wir wissen, wie eine Festplatte funktioniert.

🔧 Aufbau einer Festplatte

Festplatten-Struktur

Platter

Kleinste Speichereinheit

📐 Grundbegriffe

  • Sektor: Kleinste adressierbare Einheit (meist 512 Bytes oder 4 KB)
  • Cluster/Block: Gruppe von Sektoren, die als Einheit behandelt werden
  • Spur: Kreisförmiger Bereich auf einer Festplatte
  • Zylinder: Alle Spuren mit gleicher Position übereinander

🗂️ Partitionen

Eine Partition ist wie ein separater Bereich auf deiner Festplatte. Stell dir vor, du teilst dein Zimmer in verschiedene Bereiche auf: einen für Kleidung, einen für Bücher, einen für Elektronik.

Beispiel: Festplatte mit mehreren Partitionen

✅ Vorteile von Partitionen

  • Organisation: Trennung von System- und Benutzerdateien
  • Sicherheit: Probleme in einer Partition betreffen nicht die anderen
  • Performance: Verschiedene Dateisysteme für verschiedene Zwecke
  • Dual-Boot: Mehrere Betriebssysteme auf einer Festplatte

📋 File Allocation Table (FAT)

FAT ist eines der ältesten und einfachsten Dateisysteme. Der Name bedeutet "Datei-Zuordnungs-Tabelle" - und genau das ist es auch!

FAT-Tabelle (vereinfacht)

Cluster Nächster Cluster Inhalt
1 2 Anfang von "Brief.txt"
2 5 Fortsetzung "Brief.txt"
3 EOF "Foto.jpg" (komplette Datei)
4 FREI Nicht belegt
5 EOF Ende von "Brief.txt"

📊 FAT-Varianten

Version Max. Dateigröße Max. Partitionsgröße Besonderheiten
FAT16 2 GB 2 GB Alt, nur für kleine Medien
FAT32 4 GB 2 TB Weit verbreitet, USB-Sticks
exFAT 16 EB 128 PB Modern, große Dateien

⚠️ FAT32-Limitierung

FAT32 kann keine Dateien größer als 4 GB speichern! Das ist ein Problem bei:

  • HD-Videos (oft > 4 GB)
  • ISO-Dateien von DVDs
  • Großen Backup-Dateien

🧩 Fragmentierung

Fragmentierung ist wie ein Puzzle, bei dem die Teile über den ganzen Tisch verstreut sind. Die Datei ist vollständig, aber ihre Teile liegen nicht zusammen.

💡 Definition: Fragmentierung

Fragmentierung tritt auf, wenn eine Datei nicht in aufeinanderfolgenden Clustern gespeichert wird, sondern über verschiedene Bereiche der Festplatte verteilt ist.

⚠️ Wichtig bei SSDs

SSDs (Solid State Drives) sollten NICHT defragmentiert werden, da:

  • Kein mechanischer Kopf existiert (kein Performance-Gewinn)
  • Unnötige Schreibvorgänge die Lebensdauer verkürzen
  • SSD-Controller optimieren automatisch

🏢 NTFS (New Technology File System)

NTFS ist das moderne Dateisystem von Windows. Es ist wie ein hochmodernes Lagersystem mit vielen Sicherheitsfeatures und erweiterten Funktionen.

💡 NTFS-Features

  • Journaling: Protokolliert alle Änderungen für Crash-Sicherheit
  • Zugriffsrechte: Detaillierte Benutzer- und Gruppenrechte
  • Kompression: Automatische Dateikompression
  • Verschlüsselung: EFS (Encrypting File System)
  • Große Dateien: Bis zu 16 TB pro Datei
  • Quotas: Speicherplatz-Beschränkungen pro Benutzer

📊 NTFS vs. FAT32 Vergleich

Merkmal NTFS FAT32
Max. Dateigröße 16 TB 4 GB
Max. Partition 256 TB 2 TB
Zugriffsrechte ✅ Ja ❌ Nein
Journaling ✅ Ja ❌ Nein

🐧 EXT-Dateisysteme (Linux)

EXT (Extended File System) ist die Dateisystem-Familie von Linux. Es ist wie ein modulares Baukastensystem, das über die Jahre immer weiter verbessert wurde.

📈 EXT-Evolution

Version Jahr Max. Dateigröße Max. Partition Besonderheiten
EXT2 1993 2 TB 4 TB Kein Journaling
EXT3 2001 2 TB 4 TB + Journaling
EXT4 2008 16 TB 1 EB + Extents, bessere Performance

💡 Was macht EXT4 besonders?

  • Extents: Effizientere Speicherung großer Dateien
  • Delayed Allocation: Optimiert Schreibvorgänge
  • Online Defragmentation: Defragmentierung im laufenden Betrieb
  • Flexible Blockgrößen: 1KB bis 64KB
  • Schnelle fsck: Schnellere Dateisystem-Prüfung

🔍 Was ist eine Inode?

Eine Inode (Index Node) ist wie eine Karteikarte für jede Datei:

  • Datei-Größe und Typ
  • Zugriffsrechte
  • Zeitstempel (erstellt, geändert, zugegriffen)
  • Verweise auf die Datenblöcke
  • Anzahl der Hard Links

🔐 Zugriffsrechte

Zugriffsrechte sind wie Schlüssel zu verschiedenen Türen. Sie bestimmen, wer was mit Dateien und Ordnern machen darf.

🐧 Unix/Linux-Zugriffsrechte

Linux verwendet ein einfaches aber mächtiges System mit drei Arten von Benutzern und drei Arten von Rechten:

Zugriffsrechte-Schema

-rwxrwxrwx
Dateityp
- = Datei
d = Ordner
Owner
Besitzer der Datei
Group
Benutzergruppe
Others
Alle anderen

🔤 Rechte-Bedeutung

Recht Symbol Oktal Datei Ordner
Read r 4 Datei lesen Inhalt auflisten
Write w 2 Datei ändern Dateien erstellen/löschen
Execute x 1 Datei ausführen Ordner betreten

🔍 Beispiele für Zugriffsrechte

-rw-r--r-- dokument.txt # (644) # Owner: lesen + schreiben # Group: nur lesen # Others: nur lesen -rwxr-xr-x programm.exe # (755) # Owner: alles (lesen + schreiben + ausführen) # Group: lesen + ausführen # Others: lesen + ausführen drwxrwx--- projekt/ # (770) # Owner: alles # Group: alles # Others: nichts

🏢 Windows-Zugriffsrechte (NTFS)

Windows verwendet Access Control Lists (ACLs) - viel detaillierter als Unix:

💡 NTFS-Berechtigungen

  • Full Control: Vollzugriff (wie root)
  • Modify: Ändern (lesen + schreiben + löschen)
  • Read & Execute: Lesen und Ausführen
  • Read: Nur lesen
  • Write: Nur schreiben
  • Special Permissions: Individuelle Feineinstellungen

💻 Praxis & Befehle

🐧 Linux-Befehle

ls -la # Zeigt alle Dateien mit Rechten -rw-r--r-- 1 user group 1024 Jan 15 10:30 datei.txt df -h # Zeigt Festplatten-Belegung in menschenlesbarer Form Filesystem Size Used Avail Use% Mounted on /dev/sda1 20G 15G 4.2G 79% / chmod 755 datei.txt # Setzt Rechte auf rwxr-xr-x chown benutzer:gruppe datei.txt # Ändert Besitzer und Gruppe mount # Zeigt alle gemounteten Dateisysteme umount /dev/sda1 # Hängt Dateisystem aus fsck -f /dev/sda1 # Prüft Dateisystem auf Fehler (force) du -sh /home/user # Zeigt Verzeichnisgröße

🏢 Windows-Befehle

dir # Zeigt Ordnerinhalt chkdsk C: /f # Prüft und repariert Laufwerk C: (fix) defrag C: /O # Defragmentiert Laufwerk C: (optimize) icacls datei.txt /grant benutzer:F # Gibt vollzugriff (Full Control) fsutil fsinfo volumeinfo C: # Zeigt Dateisystem-Informationen diskpart # Erweiterte Festplatten-Verwaltung sfc /scannow # System File Checker - repariert Systemdateien

🛠️ Praktische Tipps

⚠️ Vorsicht bei Dateisystem-Operationen

  • Backup erstellen vor Partitionierung oder Formatierung
  • Niemals fsck auf gemounteten Systemen ausführen
  • Defragmentierung nur bei HDDs, nicht bei SSDs
  • Zugriffsrechte sorgfältig setzen (Sicherheit geht vor!)
  • Root-Rechte sparsam verwenden

🔍 Dateisystem wählen - Entscheidungshilfe

  • FAT32: USB-Sticks, maximale Kompatibilität wichtig
  • exFAT: Große USB-Sticks/SD-Karten, große Dateien (>4GB)
  • NTFS: Windows-Systeme, Sicherheit und erweiterte Features
  • EXT4: Linux-Systeme, beste Performance und Stabilität
  • APFS: macOS (nicht behandelt, aber erwähnenswert)

🎯 Zusammenfassung

💡 Das Wichtigste in Kürze

  • Dateisysteme organisieren und verwalten Dateien auf Speichermedien
  • Partitionen teilen Festplatten in logische, getrennte Bereiche
  • FAT ist einfach und universell kompatibel, aber funktional limitiert
  • NTFS bietet erweiterte Sicherheit und Features für Windows
  • EXT4 ist das moderne, optimierte Dateisystem für Linux
  • Zugriffsrechte schützen Dateien vor unbefugtem Zugriff
  • Fragmentierung verlangsamt nur HDDs, nicht SSDs
  • Regelmäßige Wartung hält Dateisysteme gesund und schnell
