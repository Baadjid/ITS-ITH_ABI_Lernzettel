BPE 9.1: Aufgaben und Anforderungen an Datenbankmanagementsysteme

📚 Was ist eine Datenbank?

🎯 Definition

Eine Datenbank ist eine Sammlung gespeicherter Daten, die untereinander in einer logischen Beziehung stehen und von einem Datenbankverwaltungssystem (DBMS) verwaltet werden.

Stellen Sie sich vor, Sie haben einen riesigen, gut organisierten Aktenschrank mit Millionen von Dokumenten. Eine Datenbank funktioniert ähnlich - nur digital und viel effizienter!

🏗️ Die drei wichtigsten Komponenten

DB (Datenbank) Die eigentlichen Daten - wie die Dokumente in unserem Aktenschrank
DBMS (Datenbankmanagementsystem) Die Software, die alles verwaltet - wie ein sehr intelligenter Bibliothekar
DBS (Datenbanksystem) Die Kombination aus Datenbank + DBMS + Anwenderprogramme

⭐ Zentrale Eigenschaften von Datenbanken

Was macht Datenbanken so besonders? Hier sind die vier Schlüsseleigenschaften:

🏛️

Zentrale Verwaltung

Alle Programme arbeiten mit derselben Datenbasis - keine doppelten oder widersprüchlichen Informationen mehr!

🔄

Aktualität

Daten sind für alle Anwendungen gleich aktuell. Wenn sich etwas ändert, sehen es alle sofort.

💾

Einmalige Speicherung

Vermeidung von Redundanz - jede Information wird nur einmal gespeichert und überall wiederverwendet.

👥

Gleichzeitiger Zugriff

Mehrere Benutzer können parallel arbeiten, ohne sich gegenseitig zu stören.

📋 Die 7 Hauptanforderungen an Datenbanksysteme

Ein professionelles Datenbanksystem muss diese sieben kritischen Anforderungen erfüllen:

🔄

1. Redundanzarme Speicherung

Daten möglichst nur einmal speichern - spart Platz und verhindert Widersprüche

🔍

2. Abfrage- und Änderbarkeit

Flexible Datenzugriffe ermöglichen - schnell finden und bearbeiten können

🔒

3. Logische Unabhängigkeit

Benutzer muss physische Struktur nicht kennen - einfache Bedienung

🛡️

4. Zugriffsschutz

Autorisierung und Passwörter - nur befugte Personen haben Zugang

✅

5. Integrität

Physische, logische und semantische Korrektheit - die Daten stimmen

👥

6. Mehrfachzugriff (Concurrency)

Gleichzeitige Zugriffe verwalten - viele Nutzer, kein Chaos

🔧

7. Ausfallsicherheit

Rekonstruierbarkeit aller Daten - Backup und Recovery

💡 Merkhilfe für die Prüfung

R-A-L-Z-I-M-A: Redundanzarm, Abfragbar, Logisch unabhängig, Zugriffsschutz, Integrität, Mehrfachzugriff, Ausfallsicher

🏗️ Datenbankarchitekturen

Es gibt verschiedene Wege, wie Datenbanken aufgebaut werden können. Hier die drei wichtigsten Architekturen:

Architektur Aufbau Vorteile Nachteile Anwendung
Desktop-DB (Einbenutzer) Frontend + Backend in einer Datei Einfach, kostengünstig Nur ein Benutzer Persönliche Projekte
Desktop-DB (Mehrbenutzer) Frontend-Datei + Backend auf Netzwerk Mehrere Benutzer möglich Hohe Netzwerklast Kleine Teams
Client/Server-DB Client sendet Anfragen an DB-Server Effizient, professionell Komplexer, teurer Unternehmen

🌟 Client/Server - Der Goldstandard

Bei Client/Server-Datenbanken wird nur die SQL-Anfrage und das Ergebnis über das Netzwerk geschickt - nicht die gesamten Daten! Das macht sie viel effizienter als Desktop-Datenbanken.

🏛️ Die Drei-Ebenen-Architektur

Die Drei-Ebenen-Architektur trennt die Datensicht in verschiedene Abstraktionsebenen - wie ein Hochhaus mit verschiedenen Stockwerken:

Externes Schema
Anwendersicht (Views)
↕️
Konzeptuelles Schema
Logisches Datenmodell (Basistabellen)
↕️
Internes Schema
Physische Speicherung
Externes Schema (Obergeschoss) Was der Benutzer sieht - individuelle Views und Sichten auf die Daten
Konzeptuelles Schema (Erdgeschoss) Die logische Struktur - wie Tabellen miteinander verknüpft sind
Internes Schema (Keller) Die physische Speicherung - wie die Daten auf der Festplatte liegen

🎯 Warum diese Trennung?

Jede Ebene kann verändert werden, ohne die anderen zu beeinflussen. Der Benutzer kann seine Sicht anpassen, ohne die Speicherung zu ändern!

🎯 Fokus: Relationale Datenbanken

Für das Abitur sind relationale Datenbanken am wichtigsten! Diese basieren auf Tabellen und verwenden SQL als Abfragesprache. 95% aller Prüfungsaufgaben behandeln diesen Typ!

✅ Vorteile relationaler DB

  • Leichte Änderbarkeit des Aufbaus
  • Mathematisch fundiert (Relationenalgebra)
  • Einfach programmierbar
  • Standardisiert (SQL)
  • Weit verbreitet

❌ Nachteile relationaler DB

  • Viele Ein-/Ausgaben notwendig
  • Hohe Rechnerleistung erforderlich
  • Komplex bei großen Datenmengen
  • Nicht optimal für multimediale Daten

📝 Zusammenfassung: Das Wichtigste auf einen Blick

  • Datenbank = DB + DBMS + Anwenderprogramme - drei Komponenten arbeiten zusammen
  • Vier zentrale Eigenschaften: Zentrale Verwaltung, Aktualität, einmalige Speicherung, gleichzeitiger Zugriff
  • Sieben Hauptanforderungen: R-A-L-Z-I-M-A (Redundanzarm, Abfragbar, Logisch unabhängig, Zugriffsschutz, Integrität, Mehrfachzugriff, Ausfallsicher)
  • Client/Server-Architektur ist der professionelle Standard für Unternehmen
  • Drei-Ebenen-Architektur: Externe, konzeptuelle und interne Sicht trennen
  • Relationale Datenbanken mit Tabellen und SQL sind fürs Abitur am wichtigsten

🎯 Tipps für die Abiturprüfung

⚠️ Häufige Prüfungsfragen

Typische Aufgaben:

  • Erklären Sie die drei Komponenten eines Datenbanksystems
  • Nennen und erläutern Sie fünf Anforderungen an Datenbanksysteme
  • Vergleichen Sie Desktop- und Client/Server-Architekturen
  • Beschreiben Sie die Drei-Ebenen-Architektur
  • Erläutern Sie Vor- und Nachteile relationaler Datenbanken

📚 Lernstrategie

  1. Definitionen auswendig lernen: DB, DBMS, DBS
  2. Die 7 Anforderungen mit Merkhilfe: R-A-L-Z-I-M-A
  3. Architekturen vergleichen können: Vor-/Nachteile
  4. Drei-Ebenen-Modell zeichnen können
  5. Relationale DB-Eigenschaften erklären können

💡 Musterlösung für "Anforderungen an DBMS"

Mögliche Antwort (6 Punkte):

"Ein DBMS muss folgende Anforderungen erfüllen: (1) Redundanzarme Speicherung - Daten nur einmal speichern um Inkonsistenzen zu vermeiden, (2) Flexible Abfrage- und Änderbarkeit - unterschiedliche Zugriffsmöglichkeiten auf Daten, (3) Zugriffsschutz - Benutzerautorisierung und Rechteverwaltung, (4) Integrität - Sicherstellung der Datenkonsistenz, (5) Mehrfachzugriff - gleichzeitige Nutzung durch mehrere Anwender, (6) Ausfallsicherheit - Backup und Recovery-Mechanismen."
