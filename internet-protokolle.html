🎯 Was sind Internet-Protokolle?

Die fundamentalen Regeln der Internet-Kommunikation

🤔 Stell dir vor...

Du möchtest einem Freund in einem anderen Land einen Brief schicken. Du brauchst:

  • Eine gemeinsame Sprache - damit er dich versteht
  • Eine Adresse - damit der Brief ankommt
  • Ein Transportsystem - Post, DHL, etc.
  • Regeln - wie man einen Brief formatiert

Genauso funktioniert das Internet! Computer brauchen Protokolle (= Regeln), um miteinander zu kommunizieren.

📚 Definition: Internet-Protokolle

Internet-Protokolle sind standardisierte Regeln und Verfahren, die festlegen, wie Computer und andere Geräte im Internet oder in Netzwerken miteinander kommunizieren. Sie definieren:

📝 Format
Wie Daten strukturiert werden müssen
🚚 Transport
Wie Daten übertragen werden
📍 Adressierung
Wie Geräte gefunden werden
🔧 Fehlerbehandlung
Was bei Problemen passiert
✅ Warum sind Protokolle wichtig?

Ohne Protokolle wäre das Internet wie ein Turmbau zu Babel - jeder würde seine eigene "Sprache" sprechen und niemand würde den anderen verstehen. Protokolle sorgen dafür, dass ein Windows-PC mit einem Mac, einem Smartphone und einem Server kommunizieren kann!

📚 Das TCP/IP-Schichtenmodell

Wie das Internet in Schichten organisiert ist

🏗️ Das Schichten-Prinzip

Das Internet funktioniert wie ein Schichtsystem - jede Schicht hat eine spezielle Aufgabe und arbeitet mit den anderen zusammen:

🎯 Schicht 4: Anwendungen
HTTP, HTTPS, FTP, SMTP, DNS, DHCP
🚛 Schicht 3: Transport
TCP, UDP
🌍 Schicht 2: Internet
IP (IPv4, IPv6), ICMP
🔌 Schicht 1: Netzzugang
Ethernet, WLAN, etc.
💡 Alltags-Beispiel: Eine Webseite aufrufen
  1. Anwendung: Du tippst "www.google.de" in deinen Browser
  2. Transport: TCP sorgt dafür, dass alle Daten vollständig ankommen
  3. Internet: IP findet den Weg zu Googles Servern
  4. Netzzugang: Ethernet/WLAN schickt die Daten über das Kabel/Funk

🚛 Transport-Protokolle: TCP vs UDP

Wie Daten sicher und schnell transportiert werden

🤝 TCP - Transmission Control Protocol

TCP ist wie ein zuverlässiger Paketdienst - er garantiert, dass alles ankommt!

🛡️ TCP - Der "Zuverlässige"

Eigenschaften:

  • ✅ Verbindungsorientiert: Baut eine "Leitung" auf, bevor Daten gesendet werden
  • ✅ Zuverlässig: Überprüft, ob alle Daten angekommen sind
  • ✅ Reihenfolge: Stellt sicher, dass Daten in der richtigen Reihenfolge ankommen
  • ❌ Langsamer: Viel Overhead durch Sicherheitschecks
📦 TCP-Header Struktur:
Source Port (16 Bit) Destination Port (16 Bit) Sequence Number (32 Bit) Acknowledgment (32 Bit) Header Länge Flags (SYN,ACK,FIN...) Window Size Checksum

Wichtige Felder: Source/Destination Port = Absender/Empfänger, Sequence Number = Paketnummer, ACK = Bestätigung

⚡ UDP - User Datagram Protocol

⚡ UDP - Der "Schnelle"

Eigenschaften:

  • ✅ Verbindungslos: Schickt Daten direkt ab - "Fire and Forget"
  • ✅ Schnell: Minimaler Overhead
  • ❌ Unzuverlässig: Keine Garantie, dass Daten ankommen
  • ❌ Keine Reihenfolge: Pakete können durcheinander ankommen
📦 UDP-Header Struktur (viel einfacher!):
Source Port (16 Bit) Destination Port (16 Bit) Length (16 Bit) Checksum (16 Bit)

Nur 8 Bytes Header vs TCP mit 20-60 Bytes!

⚖️ TCP vs UDP - Der große Vergleich

Eigenschaft 🛡️ TCP ⚡ UDP
Verbindung Verbindungsorientiert (wie Telefon) Verbindungslos (wie Postkarte)
Zuverlässigkeit 100% zuverlässig Best Effort (hoffentlich kommt's an)
Geschwindigkeit Langsamer (viele Checks) Schneller (keine Checks)
Header-Größe 20-60 Bytes 8 Bytes
Verwendung Webseiten, E-Mail, Dateitransfer Live-Video, Games, DNS
🎮 Wann verwende ich was?

TCP verwenden bei:

  • 📧 E-Mail (muss vollständig ankommen)
  • 🌐 Webseiten (alle HTML-Daten sind wichtig)
  • 📁 Datei-Downloads (jedes Bit muss stimmen)

UDP verwenden bei:

  • 🎮 Online-Games (Speed wichtiger als Perfektion)
  • 📹 Video-Streaming (ein verlorenes Bild ist OK)
  • 🎵 Musik-Streaming (ein kurzer Aussetzer ist OK)

🌍 Das Internet-Protokoll (IP)

Wie Daten den Weg durch das Internet finden

🗺️ IP - Das "GPS des Internets"

Das Internet Protocol (IP) ist wie ein digitales Postleitsystem - es sorgt dafür, dass Datenpakete den Weg zu ihrem Ziel finden!

🏠 IP-Adressen - Die "Hausadressen" des Internets

Jedes Gerät im Internet braucht eine eindeutige Adresse:

IPv4 Adressen
192.168.1.1
4 Zahlen von 0-255, durch Punkte getrennt
IPv6 Adressen
2001:db8::1
Längere Adressen mit Hexadezimalzahlen

📍 Arten von IP-Adressen

Typ IPv4 Beispiel IPv6 Beispiel Verwendung
🏠 Private Adressen 192.168.1.100
10.0.0.1		 fd00::1 Heimnetzwerk, Firma
🌐 Öffentliche Adressen 8.8.8.8 (Google DNS) 2001:4860:4860::8888 Internet-Server
🔄 Loopback 127.0.0.1 (localhost) ::1 Eigener Computer
🔗 Link Local 169.254.x.x fe80:: Automatische Konfiguration
📦 IPv4-Header Struktur:
Version (4 Bit) IHL (4 Bit) TOS (8 Bit) Paketlänge (16 Bit) Identifikation (16 Bit) Flags (3 Bit) Fragment Offset (13 Bit) TTL (8 Bit) Protokoll (8 Bit) Header Checksum (16 Bit) Sender IP-Adresse (32 Bit) Empfänger IP-Adresse (32 Bit)

Wichtige Felder: TTL = Time To Live (verhindert endlose Loops), Protokoll = welches Protokoll folgt (TCP=6, UDP=17)

🚀 So funktioniert IP-Routing:
  1. Paket erstellen: Dein Computer packt Daten in ein IP-Paket mit Ziel-Adresse
  2. Router-Entscheidung: Jeder Router schaut: "Kenne ich den Weg?" und leitet weiter
  3. Hop by Hop: Das Paket springt von Router zu Router bis zum Ziel
  4. TTL-Countdown: Bei jedem Hop wird TTL um 1 reduziert (verhindert endlose Schleifen)

💻 Anwendungs-Protokolle

Die Protokolle, die du täglich nutzt

🌐 HTTP/HTTPS - Das Web-Protokoll

🌐 HTTP - Hypertext Transfer Protocol

Das ist das Protokoll für Webseiten!

  • 🏠 Port 80: Standard HTTP (unverschlüsselt)
  • 🔒 Port 443: HTTPS (verschlüsselt mit TLS)
  • 📝 Request-Response: Client fragt, Server antwortet
HTTP Request Beispiel: GET /index.html HTTP/1.1 Host: www.example.com User-Agent: Mozilla/5.0... Accept: text/html HTTP Response Beispiel: HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1234 <html><body>Hello World!</body></html>

HTTP Methoden:

GET
Daten abrufen (Webseite laden)
POST
Daten senden (Formular abschicken)
PUT
Daten aktualisieren
DELETE
Daten löschen

📦 Header-Strukturen im Detail

Wie Protokoll-Header aufgebaut sind

🎯 Was sind Header?

Header sind wie Adressaufkleber auf Paketen - sie enthalten alle wichtigen Informationen für die Übertragung:

📮 Analogie: Briefpost
📮 Echter Brief:
  • Absender-Adresse
  • Empfänger-Adresse
  • Porto/Briefmarke
  • Datum
📦 Daten-Paket:
  • Source IP/Port
  • Destination IP/Port
  • Protokoll-Info
  • Prüfsumme

🧱 Header-Aufbau verschiedener Protokolle

📡 Ethernet Header (Layer 2):
Präambel (8 Bytes) Ziel-MAC (6 Bytes) Source-MAC (6 Bytes) Type (2 Bytes) Daten (46-1500 Bytes) Prüfsumme (4 Bytes)
🌍 IPv6 Header (einfacher als IPv4!):
Version (4) Traffic Class (8) Flow Label (20) - Payload Length (16) Next Header (8) Hop Limit (8) Source Address (128 Bit = 16 Bytes) Destination Address (128 Bit = 16 Bytes)

IPv6 Vorteile: Feste 40 Bytes (vs IPv4: 20-60 Bytes), viel mehr Adressen, bessere Performance

⚠️ Header-Overhead

Jede Schicht fügt ihren eigenen Header hinzu! Bei einer einfachen HTTP-Anfrage:

  • Ethernet: 18 Bytes
  • IP: 20 Bytes (IPv4) oder 40 Bytes (IPv6)
  • TCP: 20 Bytes
  • HTTP: ~200-500 Bytes

Total: ~250-600 Bytes Overhead - bevor auch nur 1 Byte Nutzdaten übertragen wird!

🔧 Praktische Beispiele & Anwendungen

Internet-Protokolle in der realen Welt

🌐 Komplettes Beispiel: Webseite aufrufen

🚀 Was passiert, wenn du "www.example.com" aufrufst?
  1. DNS-Auflösung:
    • Browser fragt DNS-Server: "Was ist die IP von www.example.com?"
    • DNS antwortet: "93.184.216.34"
    • Protokoll: DNS über UDP Port 53
  2. TCP-Verbindung aufbauen:
    • 3-Way-Handshake: SYN → SYN-ACK → ACK
    • Ziel: 93.184.216.34:80 (HTTP)
  3. HTTP-Request senden:
    GET / HTTP/1.1 Host: www.example.com User-Agent: Chrome/91.0 Accept: text/html,*/*
  4. Server-Response:
    HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1270 <!DOCTYPE html> <html>...</html>
  5. Verbindung schließen: TCP FIN → FIN-ACK

🎮 Gaming: Warum UDP?

🎮 Online-Gaming mit UDP

Warum nutzen Spiele UDP statt TCP?

  • ⚡ Geschwindigkeit: Keine Zeit für "Bestätigungen"
  • 🎯 Aktualität: Lieber neue Position als alte, aber sichere
  • 📊 Beispiel: Spielerposition jede 16ms (60 FPS)
🎯 Game-Logik:
• Position alt: (X=100, Y=200) - TCP würde warten bis bestätigt
• Position neu: (X=105, Y=205) - UDP schickt sofort, alte Position ist egal
• Resultat: Flüssigere Bewegung!

🔒 Sicherheits-Protokolle

🔒 TLS/SSL - Verschlüsselung

HTTPS = HTTP + TLS/SSL Verschlüsselung

🔒 TLS Handshake:
  1. Client: "Hallo Server, ich kann diese Verschlüsselungen"
  2. Server: "OK, nehmen wir diese + hier ist mein Zertifikat"
  3. Client prüft Zertifikat: "Bist du wirklich google.com?"
  4. Beide generieren geheimen Schlüssel
  5. Ab jetzt: Alle Daten verschlüsselt!

🏠 Heimnetzwerk-Protokolle

🏠 DHCP - Automatische IP-Vergabe

DHCP (Dynamic Host Configuration Protocol) vergibt automatisch IP-Adressen in deinem Heimnetzwerk:

📱 Handy verbindet sich mit WLAN:
  1. DHCP Discover: "Hallo, ich brauche eine IP-Adresse!"
  2. DHCP Offer: Router antwortet: "192.168.1.100 ist frei"
  3. DHCP Request: "OK, ich nehme 192.168.1.100"
  4. DHCP ACK: "Bestätigt, 24h gültig"

Zusätzlich erhält das Handy: Gateway (192.168.1.1), DNS-Server (8.8.8.8), Subnetzmaske

✅ Zusammenfassung: Internet-Protokolle
  • 🏗️ Schichten: Jede Schicht hat ihre Aufgabe (Anwendung → Transport → Internet → Netzzugang)
  • 🚛 Transport: TCP = zuverlässig, UDP = schnell
  • 🌍 Internet: IP sorgt für Adressierung und Routing
  • 💻 Anwendung: HTTP, DNS, FTP, E-Mail für konkrete Dienste
  • 📦 Header: Jede Schicht fügt Steuerungsinformationen hinzu
