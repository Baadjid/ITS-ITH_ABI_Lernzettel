🏗️ Klassen & Objekte

Die Grundbausteine der objektorientierten Programmierung

🎯 Was lernst du hier?

In diesem Kapitel lernst du die wichtigsten Konzepte der objektorientierten Programmierung kennen: Klassen und Objekte. Diese sind das Fundament von Java und vielen anderen modernen Programmiersprachen. Nach diesem Kapitel verstehst du, wie man Code strukturiert organisiert und warum das so wichtig ist!

🤔 Was sind Klassen und Objekte?

Die wichtigsten Begriffe einfach erklärt

🏗️ Denk an einen Bauplan für ein Haus!

Klasse = Der Bauplan für ein Haus. Er zeigt, wie das Haus aussehen soll: Wie viele Zimmer, welche Farbe, wo die Türen sind...

Objekt = Ein fertiges Haus, das nach diesem Bauplan gebaut wurde. Du kannst viele Häuser nach dem gleichen Plan bauen, aber jedes kann unterschiedlich aussehen (andere Farbe, andere Möbel).

Klasse (Class)

Eine Klasse ist ein Bauplan oder Template für Objekte. Sie definiert:

  • Attribute (Eigenschaften) - Was das Objekt "hat"
  • Methoden (Verhalten) - Was das Objekt "kann"
Objekt (Object)

Ein Objekt ist eine konkrete Instanz einer Klasse mit spezifischen Werten für die Attribute. Aus einer Klasse können viele verschiedene Objekte erstellt werden.

🚗 Beispiel: Auto

Klasse "Auto" könnte definieren:

  • Attribute: Marke, Farbe, Baujahr, Kilometerstand
  • Methoden: fahren(), bremsen(), hupen()

Objekte wären dann konkrete Autos:

  • Auto1: BMW, rot, 2020, 15.000 km
  • Auto2: Mercedes, blau, 2019, 25.000 km
  • Auto3: VW, weiß, 2021, 8.000 km

📚 Wichtige Grundbegriffe

Diese Begriffe musst du unbedingt verstehen

Begriff Erklärung Beispiel
Attribut Beschreibt den Zustand eines Objekts - die "Eigenschaften" oder "Daten" name, alter, farbe
Methode Beschreibt das Verhalten eines Objekts - was es "tun kann" fahren(), sprechen(), berechnen()
Konstruktor Spezielle Methode, die beim Erstellen eines Objekts aufgerufen wird new Auto("BMW", 2020)
Kapselung Interne Eigenschaften werden versteckt, Zugriff nur über bestimmte Methoden private Attribute + get/set-Methoden
Identität Jedes Objekt ist einzigartig, auch wenn es gleiche Werte hat Zwei identische Autos sind trotzdem verschiedene Objekte

💻 Deine erste Klasse in Java

Schritt für Schritt erklärt

Lass uns zusammen eine einfache Klasse "Person" erstellen:

public class Person {
    // Attribute (private = von außen nicht direkt zugreifbar)
    private String name;
    private int alter;
    private String email;
    
    // Konstruktor - wird beim Erstellen eines Objekts aufgerufen
    public Person(String name, int alter, String email) {
        this.name = name;      // "this" bezieht sich auf das aktuelle Objekt
        this.alter = alter;
        this.email = email;
    }
    
    // Standard-Konstruktor (ohne Parameter)
    public Person() {
        this.name = "Unbekannt";
        this.alter = 0;
        this.email = "keine@email.de";
    }
    
    // Methoden - was die Person "kann"
    public void vorstellen() {
        System.out.println("Hallo, ich bin " + name + " und bin " + alter + " Jahre alt.");
    }
    
    public void geburtstaghaben() {
        alter = alter + 1;
        System.out.println("Happy Birthday! Jetzt bin ich " + alter + " Jahre alt.");
    }
    
    // Getter-Methoden (um private Attribute zu lesen)
    public String getName() {
        return name;
    }
    
    public int getAlter() {
        return alter;
    }
    
    public String getEmail() {
        return email;
    }
    
    // Setter-Methoden (um private Attribute zu ändern)
    public void setName(String neuerName) {
        this.name = neuerName;
    }
    
    public void setAlter(int neuesAlter) {
        if (neuesAlter >= 0) {  // Validierung: Alter darf nicht negativ sein
            this.alter = neuesAlter;
        } else {
            System.out.println("Fehler: Alter darf nicht negativ sein!");
        }
    }
    
    public void setEmail(String neueEmail) {
        this.email = neueEmail;
    }
}
  • Klassendeklaration: public class Person - Hier wird die Klasse definiert
  • Attribute: private String name; - Die Eigenschaften der Person (private = geschützt)
  • Konstruktor: public Person(...) - Wird beim Erstellen aufgerufen, initialisiert die Werte
  • Methoden: public void vorstellen() - Was die Person tun kann
  • Getter/Setter: Sichere Zugriffe auf private Attribute

    • 🎭 Objekte erstellen und verwenden

    Aus der Klasse werden konkrete Objekte

    Jetzt können wir aus unserer Klasse "Person" konkrete Personen-Objekte erstellen:

    public class PersonTest {
    public static void main(String[] args) {
        // Objekte erstellen (Instanziierung)
        Person person1 = new Person("Anna", 25, "anna@example.com");
        Person person2 = new Person("Max", 30, "max@example.com");
        Person person3 = new Person(); // Standard-Konstruktor
        
        // Methoden aufrufen
        person1.vorstellen();  // Ausgabe: Hallo, ich bin Anna und bin 25 Jahre alt.
        person2.vorstellen();  // Ausgabe: Hallo, ich bin Max und bin 30 Jahre alt.
        
        // Geburtstag feiern
        person1.geburtstaghaben(); // Anna wird 26
        
        // Attribute über Getter lesen
        System.out.println("Name von Person 1: " + person1.getName());
        System.out.println("Alter von Person 2: " + person2.getAlter());
        
        // Attribute über Setter ändern
        person3.setName("Lisa");
        person3.setAlter(22);
        person3.setEmail("lisa@example.com");
        person3.vorstellen(); // Ausgabe: Hallo, ich bin Lisa und bin 22 Jahre alt.
        
        // Fehlerhafte Eingabe testen
        person1.setAlter(-5); // Ausgabe: Fehler: Alter darf nicht negativ sein!
    }
}

    🔍 Was passiert hier?

    1. Objekterstellung: new Person(...) erstellt ein neues Objekt
    2. Konstruktor-Aufruf: Die Werte werden automatisch gesetzt
    3. Methoden-Aufrufe: person1.vorstellen() führt die Methode aus
    4. Daten-Zugriff: Über Getter/Setter sicher auf Attribute zugreifen

    🔧 Konstruktoren im Detail

    Wie Objekte richtig initialisiert werden

    Was ist ein Konstruktor?

    Ein Konstruktor ist eine spezielle Methode, die automatisch aufgerufen wird, wenn ein neues Objekt erstellt wird. Er sorgt dafür, dass das Objekt in einen sinnvollen Anfangszustand versetzt wird.

    ⚠️ Wichtige Regeln für Konstruktoren:

    • Hat denselben Namen wie die Klasse
    • Hat keinen Rückgabetyp (auch nicht void!)
    • Wird mit new aufgerufen
    • Kann überladen werden (mehrere Versionen)
    • Initialisiert die Attribute des Objekts
    public class Auto {
    private String marke;
    private String farbe;
    private int baujahr;
    private double preis;
    
    // 1. Standard-Konstruktor (keine Parameter)
    public Auto() {
        this.marke = "Unbekannt";
        this.farbe = "weiß";
        this.baujahr = 2020;
        this.preis = 20000.0;
    }
    
    // 2. Konstruktor mit allen Parametern
    public Auto(String marke, String farbe, int baujahr, double preis) {
        this.marke = marke;
        this.farbe = farbe;
        this.baujahr = baujahr;
        this.preis = preis;
    }
    
    // 3. Konstruktor nur mit Marke und Baujahr
    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
        this.farbe = "weiß";      // Standardwert
        this.preis = 25000.0;     // Standardwert
    }
    
    // 4. Konstruktor mit Validierung
    public Auto(String marke, String farbe, int baujahr, double preis, boolean validate) {
        // Eingaben prüfen
        if (baujahr < 1886) {  // Erstes Auto wurde 1886 gebaut
            throw new IllegalArgumentException("Baujahr zu alt!");
        }
        if (preis < 0) {
            throw new IllegalArgumentException("Preis darf nicht negativ sein!");
        }
        
        this.marke = marke;
        this.farbe = farbe;
        this.baujahr = baujahr;
        this.preis = preis;
    }
    
    public void info() {
        System.out.println(marke + ", " + farbe + ", " + baujahr + ", " + preis + "€");
    }
}
    // Verschiedene Arten, Objekte zu erstellen:
public class AutoTest {
    public static void main(String[] args) {
        // Verschiedene Konstruktoren verwenden
        Auto auto1 = new Auto();                           // Standard-Konstruktor
        Auto auto2 = new Auto("BMW", "rot", 2021, 35000); // Alle Parameter
        Auto auto3 = new Auto("Mercedes", 2020);           // Nur Marke + Jahr
        
        // Ausgabe der Informationen
        auto1.info(); // Ausgabe: Unbekannt, weiß, 2020, 20000.0€
        auto2.info(); // Ausgabe: BMW, rot, 2021, 35000.0€
        auto3.info(); // Ausgabe: Mercedes, weiß, 2020, 25000.0€
    }
}

    🔐 Getter und Setter - Sichere Datenkapselung

    Warum private Attribute wichtig sind

    🏠 Denk an dein Zuhause!

    Du würdest nicht wollen, dass fremde Menschen einfach in dein Haus gehen und deine Sachen verändern können. Deshalb hast du Türen mit Schlössern. Nur du entscheidest, wer reinkommt und was verändert wird.

    Genauso ist es mit private Attributen - sie sind "verschlossen" und nur über spezielle "Türen" (Getter/Setter) zugänglich!

    Warum private Attribute?
    • Schutz: Verhindert unkontrollierte Änderungen
    • Validierung: Eingaben können geprüft werden
    • Flexibilität: Interne Struktur kann geändert werden
    • Debugging: Fehlerquellen sind leichter zu finden
    public class BankKonto {
    private String inhaber;
    private double kontostand;
    private String kontonummer;
    
    public BankKonto(String inhaber, String kontonummer) {
        this.inhaber = inhaber;
        this.kontonummer = kontonummer;
        this.kontostand = 0.0;  // Startet mit 0€
    }
    
    // GETTER - Attribute sicher lesen
    public String getInhaber() {
        return inhaber;
    }
    
    public double getKontostand() {
        return kontostand;
    }
    
    public String getKontonummer() {
        // Aus Sicherheitsgründen nur teilweise anzeigen
        return "****" + kontonummer.substring(kontonummer.length() - 4);
    }
    
    // SETTER - Attribute sicher ändern (mit Validierung!)
    public void setInhaber(String neuerInhaber) {
        if (neuerInhaber != null && !neuerInhaber.trim().isEmpty()) {
            this.inhaber = neuerInhaber;
        } else {
            System.out.println("Fehler: Inhaber-Name darf nicht leer sein!");
        }
    }
    
    // Spezielle Methoden für Kontostand (nicht einfacher Setter!)
    public boolean einzahlen(double betrag) {
        if (betrag > 0) {
            kontostand += betrag;
            System.out.println("Eingezahlt: " + betrag + "€. Neuer Kontostand: " + kontostand + "€");
            return true;
        } else {
            System.out.println("Fehler: Betrag muss positiv sein!");
            return false;
        }
    }
    
    public boolean abheben(double betrag) {
        if (betrag > 0 && betrag <= kontostand) {
            kontostand -= betrag;
            System.out.println("Abgehoben: " + betrag + "€. Neuer Kontostand: " + kontostand + "€");
            return true;
        } else if (betrag <= 0) {
            System.out.println("Fehler: Betrag muss positiv sein!");
            return false;
        } else {
            System.out.println("Fehler: Nicht genug Geld auf dem Konto!");
            return false;
        }
    }
}

    ✅ Vorteile von Getter/Setter:

    • Kontrolle: Du entscheidest, wie auf Daten zugegriffen wird
    • Validierung: Falsche Eingaben werden abgefangen
    • Sicherheit: Sensible Daten können teilweise versteckt werden
    • Flexibilität: Verhalten kann später geändert werden
    // Verwendung des BankKonto:
public class BankTest {
    public static void main(String[] args) {
        BankKonto meinKonto = new BankKonto("Max Mustermann", "1234567890");
        
        // Geld einzahlen
        meinKonto.einzahlen(100.0);    // Erfolgreich
        meinKonto.einzahlen(-50.0);    // Fehler: Betrag muss positiv sein!
        
        // Geld abheben
        meinKonto.abheben(30.0);       // Erfolgreich
        meinKonto.abheben(200.0);      // Fehler: Nicht genug Geld!
        
        // Daten sicher lesen
        System.out.println("Inhaber: " + meinKonto.getInhaber());
        System.out.println("Kontostand: " + meinKonto.getKontostand() + "€");
        System.out.println("Kontonummer: " + meinKonto.getKontonummer());
        
        // Direkter Zugriff nicht möglich (wäre ein Fehler):
        // meinKonto.kontostand = 1000000; // FEHLER! private Attribute
    }
}

    📊 Klassendiagramme - Klassen visualisieren

    Wie man Klassen grafisch darstellt

    Ein Klassendiagramm ist eine grafische Darstellung einer Klasse. Es zeigt die Struktur übersichtlich an:

    Person
    - name: String
    - alter: int
    - email: String
    + Person(name, alter, email)
    + Person()
    + vorstellen(): void
    + geburtstaghaben(): void
    + getName(): String
    + setName(name): void
    + getAlter(): int
    + setAlter(alter): void
    Aufbau eines Klassendiagramms:
    • Oben: Klassenname (fett geschrieben)
    • Mitte: Attribute mit Datentypen
    • Unten: Methoden mit Parametern und Rückgabetypen

    🔍 Symbole verstehen:

    • - (Minus) = private (nur innerhalb der Klasse sichtbar)
    • + (Plus) = public (von überall sichtbar)
    • # (Hash) = protected (in Vererbung sichtbar)
    • ~ (Tilde) = package-private (im gleichen Paket sichtbar)

    🎭 Objektdiagramme - Konkrete Objekte zeigen

    Momentaufnahmen von erstellten Objekten

    Ein Objektdiagramm zeigt konkrete Objekte mit ihren aktuellen Werten zu einem bestimmten Zeitpunkt:

    person1: Person
    name = "Anna"
    alter = 25
    email = "anna@example.com"
    person2: Person
    name = "Max"
    alter = 30
    email = "max@example.com"
    Unterschied Klassen- vs. Objektdiagramm:
    • Klassendiagramm: Zeigt die Struktur (Bauplan)
    • Objektdiagramm: Zeigt konkrete Instanzen mit Werten
    • Objektname: objektname : Klassenname (alles unterstrichen)
    • Werte: Konkrete Attributwerte werden angezeigt
    • Keine Methoden: Nur Daten, da Verhalten gleich bleibt

    Klick hier für die Lösung 
    public class Handy {
    private String marke;
    private String modell;
    private int speicher;
    private int batteriestand;
    
    public Handy(String marke, String modell, int speicher) {
        this.marke = marke;
        this.modell = modell;
        this.speicher = speicher;
        this.batteriestand = 100; // Startet voll aufgeladen
    }
    
    public void telefonieren(int minuten) {
        int verbrauch = minuten * 2;
        if (batteriestand >= verbrauch) {
            batteriestand -= verbrauch;
            System.out.println("Telefoniert für " + minuten + " Minuten. Batterie: " + batteriestand + "%");
        } else {
            System.out.println("Nicht genug Batterie! Nur noch " + batteriestand + "%");
        }
    }
    
    public void aufladen() {
        batteriestand = 100;
        System.out.println("Handy aufgeladen! Batterie: 100%");
    }
    
    public void info() {
        System.out.println("Handy: " + marke + " " + modell + ", Speicher: " + speicher + "GB, Batterie: " + batteriestand + "%");
    }
    
    // Getter
    public String getMarke() { return marke; }
    public String getModell() { return modell; }
    public int getSpeicher() { return speicher; }
    public int getBatteriestand() { return batteriestand; }
    
    // Setter
    public void setMarke(String marke) { this.marke = marke; }
    public void setModell(String modell) { this.modell = modell; }
    public void setSpeicher(int speicher) { this.speicher = speicher; }
    
    public void setBatteriestand(int batteriestand) {
        if (batteriestand >= 0 && batteriestand <= 100) {
            this.batteriestand = batteriestand;
        } else {
            System.out.println("Fehler: Batteriestand muss zwischen 0 und 100% liegen!");
        }
    }
}

    📝 Zusammenfassung

    Das Wichtigste auf einen Blick

    Das hast du gelernt:
    • ✅ Klassen sind Baupläne für Objekte
    • ✅ Objekte sind konkrete Instanzen von Klassen
    • ✅ Attribute beschreiben Eigenschaften (Was hat das Objekt?)
    • ✅ Methoden beschreiben Verhalten (Was kann das Objekt?)
    • ✅ Konstruktoren initialisieren Objekte beim Erstellen
    • ✅ Kapselung schützt Daten durch private Attribute
    • ✅ Getter/Setter ermöglichen sicheren Zugriff auf Attribute
    • ✅ Klassendiagramme visualisieren die Struktur
    • ✅ Objektdiagramme zeigen konkrete Instanzen

    🎯 Wichtige Prüfungsthemen:

    • Klassenaufbau: Attribute, Methoden, Konstruktoren
    • Objekt-Erstellung: new Klassenname(parameter)
    • Zugriffssichtbarkeiten: private, public, protected
    • this-Referenz: Bezug auf aktuelles Objekt
    • Datenkapselung: Warum private Attribute wichtig sind
    • Konstruktor-Überladung: Mehrere Konstruktoren
    • UML-Diagramme: Klassen- und Objektdiagramme lesen und erstellen

    🧠 Merkhilfe:

    Klasse = Kuchenrezept, Objekt = gebackener Kuchen

    Das Rezept (Klasse) beschreibt, wie der Kuchen gemacht wird. Der fertige Kuchen (Objekt) ist das konkrete Ergebnis. Du kannst mit einem Rezept viele verschiedene Kuchen backen!
