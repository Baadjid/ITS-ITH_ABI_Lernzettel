🎯 Was sind Konstruktoren?

Die Grundlagen der Objekterstellung verstehen

📚 Einfach erklärt:

Stell dir vor, du baust ein LEGO-Haus. Bevor du das Haus nutzen kannst, musst du es erst zusammenbauen und einrichten. Genau das macht ein Konstruktor in der Programmierung - er "baut" ein Objekt zusammen und richtet es ein, bevor es verwendet werden kann.

🔍 Definition: Konstruktor

Ein Konstruktor ist eine spezielle Methode in einer Klasse, die automatisch aufgerufen wird, wenn ein neues Objekt dieser Klasse erstellt wird. Seine Hauptaufgabe ist es, das Objekt in einen definierten Anfangszustand zu versetzen.

🔧 Hauptfunktionen

  • Objekt initialisieren
  • Speicher reservieren
  • Startwerte setzen
  • Validierung durchführen

⚡ Besondere Eigenschaften

  • Gleicher Name wie Klasse
  • Kein Rückgabetyp
  • Automatischer Aufruf
  • Kann überladen werden

📝 Aufbau und Syntax eines Konstruktors

Wie schreibt man einen Konstruktor richtig?

1

Grundaufbau

Ein Konstruktor hat immer den gleichen Namen wie die Klasse und keinen Rückgabetyp (auch nicht void).

2

Sichtbarkeitsmodifikator

Meist public, damit Objekte von außerhalb erstellt werden können.

3

Parameter

Können Parameter haben, um Startwerte zu übergeben (optional).

📄 Grundsyntax eines Konstruktors 
public class KlassenName {
    // Attribute
    private String attribut;
    
    // Konstruktor - gleicher Name wie Klasse!
    public KlassenName() {
        // Initialisierung hier
    }
}

🎭 Verschiedene Arten von Konstruktoren

Standard, parametrisiert und überladene Konstruktoren

1. Standard-Konstruktor (Default-Konstruktor)

🔍 Standard-Konstruktor

Ein Konstruktor ohne Parameter, der Standardwerte setzt. Wird automatisch vom Compiler erstellt, wenn kein Konstruktor definiert ist.

📄 Beispiel: Standard-Konstruktor 
public class Person {
    private String name;
    private int alter;
    
    // Standard-Konstruktor
    public Person() {
        this.name = "Unbekannt";
        this.alter = 0;
        System.out.println("Person erstellt mit Standardwerten");
    }
    
    // Getter-Methoden
    public String getName() {
        return name;
    }
    
    public int getAlter() {
        return alter;
    }
}

2. Parametrisierter Konstruktor

🔍 Parametrisierter Konstruktor

Ein Konstruktor mit Parametern, der es ermöglicht, beim Erstellen des Objekts spezifische Werte zu übergeben.

📄 Beispiel: Parametrisierter Konstruktor 
public class Auto {
    private String marke;
    private int baujahr;
    private double preis;
    
    // Parametrisierter Konstruktor
    public Auto(String marke, int baujahr, double preis) {
        this.marke = marke;
        this.baujahr = baujahr;
        this.preis = preis;
        System.out.println("Auto erstellt: " + marke + " (" + baujahr + ") für " + preis + "€");
    }
    
    // Objekterstellung
    public static void main(String[] args) {
        Auto meinAuto = new Auto("BMW", 2020, 35000.50);
    }
}

3. Konstruktor-Überladung

🔍 Konstruktor-Überladung

Mehrere Konstruktoren in einer Klasse mit unterschiedlichen Parameterlisten. Ermöglicht flexible Objekterstellung.

📄 Beispiel: Überladene Konstruktoren 
public class Rechteck {
    private double breite;
    private double hoehe;
    private String farbe;
    
    // 1. Standard-Konstruktor
    public Rechteck() {
        this.breite = 1.0;
        this.hoehe = 1.0;
        this.farbe = "weiß";
    }
    
    // 2. Konstruktor mit Größe
    public Rechteck(double breite, double hoehe) {
        this.breite = breite;
        this.hoehe = hoehe;
        this.farbe = "weiß"; // Standardfarbe
    }
    
    // 3. Konstruktor mit allem
    public Rechteck(double breite, double hoehe, String farbe) {
        this.breite = breite;
        this.hoehe = hoehe;
        this.farbe = farbe;
    }
    
    // 4. Quadrat-Konstruktor (nur eine Seitenlänge)
    public Rechteck(double seitenlaenge) {
        this(seitenlaenge, seitenlaenge); // Aufruf eines anderen Konstruktors!
    }
    
    public double berechneFlaeche() {
        return breite * hoehe;
    }
}

🛡️ Konstruktor mit Validierung

Sichere Objekterstellung durch Eingabeprüfung

⚠️ Wichtig: Eingabevalidierung

Konstruktoren sollten immer prüfen, ob die übergebenen Werte sinnvoll sind. Ungültige Werte können zu Fehlern im Programm führen!

📄 Beispiel: Konstruktor mit Validierung 
public class BankKonto {
    private String kontoinhaber;
    private double kontostand;
    private String iban;
    
    public BankKonto(String kontoinhaber, double kontostand, String iban) {
        // Validierung des Kontoinhabers
        if (kontoinhaber == null || kontoinhaber.trim().isEmpty()) {
            throw new IllegalArgumentException("Kontoinhaber darf nicht leer sein!");
        }
        
        // Validierung des Kontostands
        if (kontostand < 0) {
            throw new IllegalArgumentException("Kontostand darf nicht negativ sein!");
        }
        
        // Validierung der IBAN (vereinfacht)
        if (iban == null || iban.length() != 22) {
            throw new IllegalArgumentException("IBAN muss 22 Zeichen lang sein!");
        }
        
        // Erst nach erfolgreicher Validierung zuweisen
        this.kontoinhaber = kontoinhaber.trim();
        this.kontostand = kontostand;
        this.iban = iban.toUpperCase();
        
        System.out.println("Bankkonto erfolgreich erstellt für: " + this.kontoinhaber);
    }
}

💡 Praktisches Beispiel: Fehlerbehandlung

📄 Verwendung mit Fehlerbehandlung 
public static void main(String[] args) {
    try {
        // Gültiges Konto erstellen
        BankKonto konto1 = new BankKonto("Max Mustermann", 1500.0, "DE89370400440532013000");

🔗 Das this-Schlüsselwort

Selbstreferenz in Konstruktoren verstehen

🔍 this-Schlüsselwort

this ist eine Referenz auf das aktuelle Objekt. Es wird verwendet, um zwischen Parametern und Instanzvariablen zu unterscheiden oder andere Konstruktoren aufzurufen.

1. this für Attributzugriff

📄 Beispiel: this bei Namenskonflikten 
public class Student {
    private String name;
    private int matrikelnummer;
    
    public Student(String name, int matrikelnummer) {
        // Ohne 'this' würde der Parameter sich selbst zugewiesen!
        this.name = name;                // this.name = Attribut, name = Parameter
        this.matrikelnummer = matrikelnummer;  // Eindeutige Zuordnung
    }
    
    // Falsche Variante (ohne this):
    // name = name;  ← Parameter wird sich selbst zugewiesen, Attribut bleibt unverändert!
}

2. this() für Konstruktor-Verkettung

📄 Beispiel: Konstruktor-Verkettung mit this() 
public class Produkt {
    private String name;
    private double preis;
    private String kategorie;
    private boolean verfuegbar;
    
    // Haupt-Konstruktor mit allen Parametern
    public Produkt(String name, double preis, String kategorie, boolean verfuegbar) {
        this.name = name;
        this.preis = preis;
        this.kategorie = kategorie;
        this.verfuegbar = verfuegbar;
        System.out.println("Vollständiges Produkt erstellt: " + name);
    }
    
    // Vereinfachter Konstruktor - ruft den Haupt-Konstruktor auf
    public Produkt(String name, double preis) {
        this(name, preis, "Allgemein", true); // Aufruf des anderen Konstruktors
    }
    
    // Standard-Konstruktor
    public Produkt() {
        this("Unbekanntes Produkt", 0.0); // Aufruf des 2-Parameter-Konstruktors
    }
}
⚠️ Wichtige Regel für this()

this() muss immer die erste Anweisung im Konstruktor sein! Davor dürfen keine anderen Anweisungen stehen.

🗑️ Destruktoren in Java

Warum Java keine expliziten Destruktoren hat

⚠️ Wichtiger Hinweis

Java hat keine expliziten Destruktoren wie C++! Die Speicherverwaltung übernimmt der Garbage Collector automatisch.

🔍 Was ist ein Destruktor?

In anderen Programmiersprachen (wie C++) ist ein Destruktor eine spezielle Methode, die automatisch aufgerufen wird, wenn ein Objekt zerstört wird. Sie räumt Ressourcen auf und gibt Speicher frei.

Sprache Destruktor-Konzept Speicherverwaltung
C++ Explizite Destruktoren (~KlassenName) Manuell (new/delete)
Java Keine Destruktoren, aber finalize() Automatisch (Garbage Collector)
C# Destruktoren (~KlassenName) und Dispose() Automatisch + manuell möglich

Java-Alternative: finalize() Methode

📄 Beispiel: finalize() Methode (VERALTET!) 
public class RessourcenKlasse {
    private String ressourcenName;
    
    public RessourcenKlasse(String name) {
        this.ressourcenName = name;
        System.out.println("Ressource erstellt: " + name);
    }
    
    // finalize() - wird vom Garbage Collector aufgerufen (NICHT EMPFOHLEN!)
    @Override
    protected void finalize() throws Throwable {
        try {
            System.out.println("Ressource wird freigegeben: " + ressourcenName);
            // Aufräumarbeiten hier...
        } finally {
            super.finalize();
        }
    }
}
⚠️ finalize() ist veraltet!

Die finalize()-Methode ist seit Java 9 als deprecated (veraltet) markiert und sollte nicht mehr verwendet werden, da sie unvorhersagbar ist und Performanceprobleme verursachen kann.

Moderne Alternative: try-with-resources

📄 Beispiel: AutoCloseable Interface (EMPFOHLEN) 
public class DateiManager implements AutoCloseable {
    private String dateiname;
    private boolean istOffen;
    
    public DateiManager(String dateiname) {
        this.dateiname = dateiname;
        this.istOffen = true;
        System.out.println("Datei geöffnet: " + dateiname);
    }
    
    public void schreiben(String inhalt) {
        if (!istOffen) {
            throw new IllegalStateException("Datei ist bereits geschlossen");
        }
        System.out.println("Schreibe in " + dateiname + ": " + inhalt);
    }
    
    // AutoCloseable Interface - automatisches Aufräumen
    @Override
    public void close() {
        if (istOffen) {
            System.out.println("Datei geschlossen: " + dateiname);
            istOffen = false;
        }
    }
    
    // Verwendung mit try-with-resources
    public static void main(String[] args) {
        // Automatisches Schließen nach dem try-Block!
        try (DateiManager datei = new DateiManager("test.txt")) {
            datei.schreiben("Hallo Welt!");
            datei.schreiben("Noch mehr Text...");
        } // close() wird automatisch aufgerufen!
        
        System.out.println("Datei wurde automatisch geschlossen");
    }
}

🚗 Vollständiges Beispiel: Auto-Klasse

Alle Konstruktor-Konzepte in der Praxis

📄 Komplette Auto-Klasse mit verschiedenen Konstruktoren 
public class Auto {
    // Attribute
    private String marke;
    private String modell;
    private int baujahr;
    private double preis;
    private String farbe;
    private int kilometerstand;
    
    // Standard-Konstruktor
    public Auto() {
        this("Unbekannt", "Unbekannt", 2024, 0.0, "Weiß", 0);
        System.out.println("Standard-Auto erstellt");
    }
    
    // Konstruktor mit Grunddaten
    public Auto(String marke, String modell) {
        this(marke, modell, 2024, 0.0, "Weiß", 0);
    }
    
    // Konstruktor mit Marke, Modell und Baujahr
    public Auto(String marke, String modell, int baujahr) {
        this(marke, modell, baujahr, 0.0, "Weiß", 0);
    }
    
    // Vollständiger Konstruktor mit Validierung
    public Auto(String marke, String modell, int baujahr, 
               double preis, String farbe, int kilometerstand) {
        
        // Validierung der Eingaben
        if (marke == null || marke.trim().isEmpty()) {
            throw new IllegalArgumentException("Marke darf nicht leer sein");
        }
        
        if (modell == null || modell.trim().isEmpty()) {
            throw new IllegalArgumentException("Modell darf nicht leer sein");
        }
        
        if (baujahr < 1886 || baujahr > 2025) { // 1886: Erstes Auto von Benz
            throw new IllegalArgumentException("Ungültiges Baujahr: " + baujahr);
        }
        
        if (preis < 0) {
            throw new IllegalArgumentException("Preis darf nicht negativ sein");
        }
        
        if (kilometerstand < 0) {
            throw new IllegalArgumentException("Kilometerstand darf nicht negativ sein");
        }
        
        // Zuweisungen nach erfolgreicher Validierung
        this.marke = marke.trim();
        this.modell = modell.trim();
        this.baujahr = baujahr;
        this.preis = preis;
        this.farbe = farbe != null ? farbe.trim() : "Weiß";
        this.kilometerstand = kilometerstand;
        
        System.out.println("Auto erstellt: " + this.marke + " " + this.modell + 
                          " (" + this.baujahr + ") - " + this.farbe);
    }
    
    // Getter-Methoden
    public String getMarke() { return marke; }
    public String getModell() { return modell; }
    public int getBaujahr() { return baujahr; }
    public double getPreis() { return preis; }
    public String getFarbe() { return farbe; }
    public int getKilometerstand() { return kilometerstand; }
    
    // Setter-Methoden mit Validierung
    public void setPreis(double preis) {
        if (preis < 0) {
            throw new IllegalArgumentException("Preis darf nicht negativ sein");
        }
        this.preis = preis;
    }
    
    // Methoden
    public void fahren(int kilometer) {
        if (kilometer > 0) {
            this.kilometerstand += kilometer;
            System.out.println(marke + " " + modell + " ist " + kilometer + " km gefahren. Neuer Stand: " + 
                              this.kilometerstand + " km");
        }
    }
    
    public int getAlter() {
        return 2024 - baujahr;
    }
    
    @Override
    public String toString() {
        return "Auto{" +
                "marke='" + marke + '\'' +
                ", modell='" + modell + '\'' +
                ", baujahr=" + baujahr +
                ", preis=" + preis +
                ", farbe='" + farbe + '\'' +
                ", kilometerstand=" + kilometerstand +
                '}';
    }
}

📚 Wichtige Punkte für die Abiturprüfung

Was du unbedingt wissen musst

🎯 Prüfungsrelevante Konzepte

  • Konstruktor-Syntax: Gleicher Name wie Klasse, kein Rückgabetyp
  • this-Schlüsselwort: Unterscheidung Parameter/Attribut, Konstruktor-Verkettung
  • Konstruktor-Überladung: Verschiedene Parameterlisten möglich
  • Validierung: Eingabeprüfung mit IllegalArgumentException
  • Standard vs. parametrisiert: Unterschiede und Anwendungsfälle
  • Objekterstellung: new-Operator ruft Konstruktor auf
  • Java-Besonderheit: Keine expliziten Destruktoren, Garbage Collector
⚠️ Häufige Prüfungsfehler vermeiden
  • Rückgabetyp bei Konstruktor: Konstruktoren haben KEINEN Rückgabetyp!
  • this() Position: this() muss die ERSTE Anweisung im Konstruktor sein
  • Parameter vs. Attribut: Ohne this. wird der Parameter sich selbst zugewiesen
  • Validierung vergessen: Immer Eingaben prüfen, besonders bei kritischen Werten
  • Exception-Handling: IllegalArgumentException für ungültige Parameter verwenden

💡 Aufgabe 2: Fehler finden

📄 Fehlerhafter Code - Was ist falsch? 
public class Person {
    private String name;
    private int alter;
    
    // Fehler 1: Rückgabetyp beim Konstruktor
    public void Person(String name) {
        // Fehler 2: this() nicht an erster Stelle
        System.out.println("Person wird erstellt");
        this("Unbekannt", 0);
    }
    
    public Person(String name, int alter) {
        // Fehler 3: Parameter wird sich selbst zugewiesen
        name = name;
        alter = alter;
    }
}
✅ Lösung der Fehler
  • Fehler 1: void entfernen - Konstruktoren haben keinen Rückgabetyp
  • Fehler 2: this() muss die erste Anweisung sein
  • Fehler 3: this.name = name; und this.alter = alter; verwenden

📋 Zusammenfassung

Das Wichtigste auf einen Blick

🔧 Was ist ein Konstruktor?

Spezielle Methode zur Initialisierung von Objekten. Wird automatisch beim Erstellen mit new aufgerufen.

📝 Syntax-Regeln

Gleicher Name wie Klasse, kein Rückgabetyp, kann Parameter haben, meist public.

🎭 Arten von Konstruktoren

Standard (ohne Parameter), parametrisiert (mit Parametern), überladen (mehrere verschiedene).

🔗 this-Schlüsselwort

Referenz auf aktuelles Objekt. Für Attributzugriff (this.name) und Konstruktor-Verkettung (this()).

🛡️ Validierung

Eingaben in Konstruktoren prüfen und bei Fehlern IllegalArgumentException werfen.

🗑️ Destruktoren

Java hat keine expliziten Destruktoren. Garbage Collector übernimmt automatisch die Speicherverwaltung.

🎯 Merksätze für die Prüfung
  • "Konstruktor = Bauanleitung für Objekte" - Setzt Anfangszustand fest
  • "this() immer zuerst!" - Konstruktor-Verkettung nur als erste Anweisung
  • "Prüfen vor Zuweisen" - Validierung verhindert fehlerhafte Objekte
  • "Java räumt automatisch auf" - Garbage Collector statt Destruktoren
  • "new ruft Konstruktor auf" - Objekterstellung und Initialisierung in einem
