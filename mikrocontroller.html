📚 Was ist ein Mikrocontroller?

🎯 Definition: Mikrocontroller (MC)

Ein Mikrocontroller ist ein vollständiger kleiner Computer auf einem einzigen Chip. Er enthält alle notwendigen Komponenten eines Computers:

  • CPU (Prozessor) - das "Gehirn" für Berechnungen
  • Speicher - für Programme und Daten
  • Ein-/Ausgabe-Ports - für Verbindungen zur Außenwelt
  • Peripherie - Timer, Kommunikationsschnittstellen, etc.

🔍 Unterschied zwischen CPU und Mikrocontroller

Eigenschaft CPU (Computer-Prozessor) Mikrocontroller

🌟 Praktische Anwendungen von Mikrocontrollern

  • Haushaltsgeräte: Waschmaschinen, Mikrowellen, Kaffeemaschinen
  • Automotive: Motorsteuerung, ABS, Airbag-Systeme
  • IoT-Geräte: Smart Home, Sensoren, Überwachungssysteme
  • Medizintechnik: Herzschrittmacher, Blutzuckermessgeräte
  • Industrieautomation: Robotersteuerung, Fertigungsanlagen

🧩 Die Hauptkomponenten im Detail

1. CPU (Central Processing Unit)

Die CPU ist das Herzstück des Mikrocontrollers und besteht aus:

  • ALU (Arithmetic Logic Unit) - führt Berechnungen durch
  • Steuerwerk - koordiniert alle Abläufe
  • Register - schnelle Zwischenspeicher
  • Instruction Decoder - entschlüsselt Programmbefehle

2. Speicher-Subsystem

  • ROM/Flash - speichert das Programm dauerhaft
  • RAM - speichert temporäre Daten und Variablen
  • EEPROM - speichert Konfigurationsdaten dauerhaft

3. Bus-System

  • Adressbus - bestimmt, wo Daten gespeichert/gelesen werden
  • Datenbus - transportiert die eigentlichen Informationen
  • Steuerbus - koordiniert Lese-/Schreibvorgänge

4. Peripherie-Module

  • GPIO - digitale Ein-/Ausgänge
  • Timer/Counter - Zeitmessung und PWM
  • Kommunikation - UART, I²C, SPI
  • Wandler - A/D und D/A Konverter

🧠 CPU-Aufbau im Detail

⚡ Das Rechenwerk (ALU)

ALU - Arithmetic Logic Unit

Die ALU ist die "Rechenzentrale" des Mikrocontrollers. Sie kann verschiedene Operationen durchführen:

🔧 ALU-Funktionen

Eine typische 32-Bit ALU besteht aus:

  • 32-Bit Volladdierer - für Addition
  • 32-Bit Subtrahierer - für Subtraktion
  • 32-Bit UND-Gatter - für logische UND-Verknüpfung
  • 32-Bit ODER-Gatter - für logische ODER-Verknüpfung
  • 32-Bit NICHT-Gatter - für Negation
  • NOP-Funktion - Durchleitung ohne Operation

Funktionsweise: Beide Operanden werden gleichzeitig mit allen Recheneinheiten verbunden. Ein Multiplexer wählt je nach Anweisung das passende Rechenergebnis aus.

Assembly-Beispiele für ALU-Operationen
; Addition ohne Flags ADD R5, R7, R6 ; R5 = R7 + R6 ; Addition mit Status-Flags ADDS R5, R7, R6 ; R5 = R7 + R6, Flags werden gesetzt ; Subtraktion mit Flags SUBS R5, R7, R6 ; R5 = R7 - R6, Flags werden gesetzt ; Laden einer Konstante LDR R7, =0x23 ; Lade Wert 0x23 in Register R7

🚩 Status-Register (Program Status Register - PSR)

Das PSR enthält wichtige Informationen über das Ergebnis der letzten Operation:

N-Flag Negative Flag - Ergebnis ist negativ
Z-Flag Zero Flag - Ergebnis ist Null
C-Flag Carry Flag - Übertrag bei Rechnung
V-Flag Overflow Flag - Zahlenbereich überschritten

💡 Beispiel: Schleifenprogramm

Dieses Programm zählt von 0 bis zu einem bestimmten Wert:

Assembly-Schleifenprogramm
main: LDR R0, =0 ; Startwert 0 in R0 laden schleife: ; Sprungmarke ADD R0, R0, #1 ; Addiere 1 zum Startwert CMP R0, #10 ; Vergleiche R0 mit 10 BNE schleife ; Springe zu "schleife" wenn nicht gleich ; Ende der Schleife - R0 enthält jetzt 10

💾 Speicherarten in Mikrocontrollern

🔄 RAM - Random Access Memory

Was ist RAM?

RAM (Random Access Memory) ist der Arbeitsspeicher des Mikrocontrollers. Charakteristika:

  • Flüchtig - Daten gehen bei Spannungsausfall verloren
  • Schnell - direkter Zugriff auf jede Speicherzelle
  • Beschreibbar - Daten können jederzeit geändert werden
SRAM (Static RAM)
✅ Vorteile
  • Sehr schnell
  • Kein Refresh nötig
  • Niedrige Latenz
❌ Nachteile
  • Großer Platzbedarf
  • Teuer
  • Niedrige Kapazität

Speicherprinzip: Daten werden in Flipflops gespeichert

DRAM (Dynamic RAM)
✅ Vorteile
  • Hohe Kapazität
  • Kleiner Platzbedarf
  • Günstig
❌ Nachteile
  • Langsamer
  • Refresh-Zyklen nötig
  • Während Refresh nicht verfügbar

Speicherprinzip: Daten werden in Kondensatoren gespeichert

📚 ROM - Read-Only Memory

Was ist ROM?

ROM (Read-Only Memory) ist der Programmspeicher. Charakteristika:

  • Nicht-flüchtig - Daten bleiben ohne Spannung erhalten
  • Dauerhaft - speichert das Hauptprogramm
  • Nur lesbar - kann im normalen Betrieb nicht verändert werden
Statischer ROM

Programmierung: Einmalig bei der Herstellung

Änderung: Nicht möglich

Verwendung: Massenproduktion mit festem Code

PROM

Programmierung: Einmalig vom Benutzer

Änderung: Nicht möglich

Verwendung: Kleine Serien, Prototyping

EPROM

Programmierung: Elektrisch programmierbar

Löschung: UV-Licht (kompletter Chip)

Verwendung: Entwicklung, mehrfache Nutzung

EEPROM

Programmierung: Elektrisch programmierbar

Löschung: Elektrisch, byteweise

Verwendung: Konfigurationsdaten, Parameter

Flash-ROM

Programmierung: Elektrisch, blockweise

Löschung: Elektrisch, blockweise

Verwendung: Moderne Mikrocontroller, häufigste Art

⚠️ Wichtiger Hinweis zu Speichergrößen

Typische Speichergrößen bei Mikrocontrollern:

  • Flash-ROM: 32KB - 2MB (Programmspeicher)
  • RAM: 2KB - 256KB (Arbeitsspeicher)
  • EEPROM: 1KB - 64KB (Konfigurationsdaten)

Zum Vergleich: Ein moderner PC hat 8-32GB RAM!

🔍 Funktionsweise

Bei der Von-Neumann-Architektur teilen sich Programme und Daten einen gemeinsamen Speicher und Bus. Dies führt zu einem "Flaschenhals", da Befehle und Daten nacheinander übertragen werden müssen.

✅ Vorteile
  • Gleichzeitiger Zugriff

❌ Nachteile
  • Komplexerer Aufbau
  • Feste Speicheraufteilung
  • Teurer

🔍 Funktionsweise

Die Harvard-Architektur verwendet getrennte Speicher und separate Busse für Programme und Daten. Dadurch können Befehle und Daten gleichzeitig übertragen werden, was zu höherer Performance führt.

🎯 Welche Architektur verwenden moderne Mikrocontroller?

Die meisten modernen Mikrocontroller (wie ARM Cortex-M, AVR, PIC) verwenden eine modifizierte Harvard-Architektur:

  • Getrennte Caches für Befehle und Daten (Harvard-Prinzip)
  • Gemeinsamer Hauptspeicher (Von-Neumann-Prinzip)
  • Beste Kombination aus Performance und Flexibilität

🔧 RISC vs. CISC

Eigenschaft RISC (Reduced Instruction Set) CISC (Complex Instruction Set)
Befehlsanzahl Wenige, einfache Befehle Viele, komplexe Befehle
Befehlslänge Konstant (z.B. 32 Bit) Variabel (1-15 Bytes)
Ausführungszeit 1 Taktzyklus pro Befehl Mehrere Taktzyklen pro Befehl
Speicherzugriff Load-Store-Architektur Direkter Speicherzugriff in Befehlen
Beispiele ARM, MIPS, RISC-V x86, x86-64

💡 Load-Store-Architektur (RISC)

Bei RISC-Prozessoren können arithmetische Operationen nur mit Registern durchgeführt werden:

RISC: Load-Store Beispiel
; Laden der Werte aus dem Speicher in Register LDR R1, [address1] ; Lade Wert von address1 in R1 LDR R2, [address2] ; Lade Wert von address2 in R2 ; Arithmetische Operation nur mit Registern ADD R3, R1, R2 ; R3 = R1 + R2 ; Ergebnis zurück in den Speicher schreiben STR R3, [result] ; Speichere R3 an Adresse result
CISC: Direkter Speicherzugriff
; Bei CISC kann direkt mit Speicherwerten gerechnet werden ADD [result], [address1], [address2] ; Direkte Addition von Speicherwerten

📋 Register-System

Was sind Register?

Register sind die schnellsten Speicher im Mikrocontroller. Sie befinden sich direkt in der CPU und können in einem Taktzyklus gelesen oder beschrieben werden.

🏗️ Aufbau eines Registers

🔧 Technische Umsetzung

Ein Register besteht aus mehreren D-Flipflops:

  • 1 Bit = 1 D-Flipflop
  • 32-Bit Register = 32 D-Flipflops
  • Multiplexer vor jedem D-FF ermöglicht Datenweiterleitung
  • Bei steigender Taktflanke wird das Signal am D-Eingang gespeichert

📊 ARM Cortex-M Register (Beispiel STM32)

General Purpose Register (R0-R15)

R0-R12 Allzweck-Register für Datenoperationen (Load-Store-Arithmetic-Logic)
R13 (SP) Stack Pointer - zeigt auf die aktuelle Position im Stack
R14 (LR) Link Register - speichert Rücksprungadresse bei Funktionsaufrufen
R15 (PC) Program Counter - zeigt auf den nächsten auszuführenden Befehl

Spezial-Register

PSR Program Status Register - enthält Flags und Systemstatus
PRIMASK Exception Mask Register - steuert Interrupt-Verhalten
CONTROL Control Register - steuert Privilegien und Stack-Auswahl

💡 Datenbreiten bei ARM

📏 ARM Datentypen

ARM-Prozessoren verwenden folgende Bezeichnungen:

  • Byte: 8 Bit (wie üblich)
  • Halfword: 16 Bit (bei x86 = 1 Word)
  • Word: 32 Bit (bei x86 = Double Word)

Achtung: Andere Prozessorfamilien haben andere Konventionen!

🔧 Register-Beispiel in Assembly

ARM Assembly - Register verwenden
; GPIO-Port verwenden LDR R3, =GPIOB ; Lade GPIOB-Adresse in Register R3 LDRB R6, [R3, #IDR] ; Lade Byte vom Input Data Register in R6 ADD R1, R6, #5 ; Addiere Konstante 5 zu R6, Ergebnis in R1 ; Anderes GPIO-Port LDR R3, =GPIOC ; Lade GPIOC-Adresse in R3 (überschreibt alten Wert!) STRB R1, [R3, #ODR] ; Speichere Byte aus R1 in Output Data Register

🔌 Peripherie-Module

📡 GPIO - General Purpose Input/Output

Was ist GPIO?

GPIO sind die "Tore zur Außenwelt" des Mikrocontrollers. Jeder Pin kann als Eingang oder Ausgang konfiguriert werden.

🏗️ GPIO-Register

Jeder GPIO-Port hat mehrere Register:

  • IDR (Input Data Register): Liest den aktuellen Zustand der Eingänge
  • ODR (Output Data Register): Steuert den Zustand der Ausgänge
  • DDR (Data Direction Register): Bestimmt ob Pin Ein- oder Ausgang ist
GPIO Programmierung
// Pin als Ausgang konfigurieren pinMode(13, OUTPUT); // LED einschalten (HIGH = 5V) digitalWrite(13, HIGH); // LED ausschalten (LOW = 0V) digitalWrite(13, LOW); // Pin als Eingang konfigurieren pinMode(2, INPUT); // Digitalen Wert lesen int sensorValue = digitalRead(2);

⏱️ Timer und PWM

Timer-Module

Timer sind sehr vielseitige Module, die für verschiedene Aufgaben verwendet werden können:

  • Zeitmessung - Präzise Zeitintervalle
  • PWM-Erzeugung - Pulsweitenmodulation für analoge Ausgaben
  • Frequenzmessung - Messen von Signalfrequenzen
  • Interrupts - Zeitgesteuerte Unterbrechungen

PWM-Signal (Pulsweitenmodulation)

      ───┘ └───┘ └───┘ └───┘ └───

         <-- T --> 75% EIN, 25% AUS

Anwendung: Helligkeit von LEDs, Geschwindigkeit von Motoren, analoge Spannungen simulieren

🌐 Kommunikationsschnittstellen

📞 UART/USART

Universal Asynchronous Receiver Transmitter
  • Serielle Kommunikation - ein Bit nach dem anderen
  • Asynchron - kein gemeinsamer Takt, Start-/Stop-Bits
  • Punkt-zu-Punkt - zwischen zwei Geräten
  • Anwendung: PC-Verbindung, GPS-Module, Bluetooth

🔄 I²C (Inter-Integrated Circuit)

Serieller Bus für kurze Distanzen
  • 2 Leitungen: SDA (Daten) und SCL (Takt)
  • Master-Slave-Prinzip: Ein Master, mehrere Slaves
  • Adressierung: Jeder Slave hat eigene 7-Bit-Adresse
  • Anwendung: Sensoren, EEPROMs, Displays

⚡ SPI (Serial Peripheral Interface)

Schnelle serielle Schnittstelle
  • 4 Leitungen: MOSI, MISO, SCK, CS
  • Vollduplex: Gleichzeitig senden und empfangen
  • Höhere Geschwindigkeit als I²C
  • Anwendung: SD-Karten, Flash-Speicher, Displays

🔄 A/D und D/A Wandler

Analog-Digital-Wandlung

Mikrocontroller sind digitale Geräte, aber die reale Welt ist analog. Wandler ermöglichen die Brücke zwischen beiden Welten:

Wandler-Typ Funktion Beispiele
A/D-Wandler (ADC) Analoge Spannung → Digitaler Wert Temperatursensoren, Potentiometer, Mikrofone
D/A-Wandler (DAC) Digitaler Wert → Analoge Spannung Audio-Ausgabe, Spannungsregelung

💡 ADC-Beispiel

12-Bit ADC bei 3.3V Referenzspannung:

  • Auflösung: 2¹² = 4096 Stufen
  • Kleinste messbare Spannung: 3.3V / 4096 = 0.8mV
  • Digitalwert 2048 entspricht: 1.65V (Mitte des Bereichs)

🔧 Was braucht ein Mikrocontroller zum Laufen?

#include // LCD initialisieren (RS, Enable, D4, D5, D6, D7) LiquidCrystal lcd(12, 11, 5, 4, 3, 2); void setup() { lcd.begin(16, 2); // 16x2 LCD initialisieren lcd.print("Temperatur:"); // Anfangstext } void loop() { // Analogwert vom Temperatursensor lesen (Pin A0) int sensorValue = analogRead(A0); // Umrechnung in Temperatur (für LM35: 10mV/°C) float voltage = sensorValue * (5.0 / 1024.0); // Spannung berechnen float temperature = voltage * 100.0; // Temperatur in °C // Temperatur auf LCD anzeigen lcd.setCursor(0, 1); // Zweite Zeile lcd.print(temperature); // Temperaturwert lcd.print(" C"); // Einheit delay(1000); // 1 Sekunde warten }

🚦 Beispiel 2: Ampelsteuerung mit Timer

Arduino Code - Ampelsteuerung
// Pin-Definitionen const int ROT = 13; const int GELB = 12; const int GRUEN = 11; // Zeiten in Millisekunden const unsigned long GRUEN_ZEIT = 5000; // 5 Sekunden const unsigned long GELB_ZEIT = 2000; // 2 Sekunden const unsigned long ROT_ZEIT = 3000; // 3 Sekunden unsigned long letzteSchaltung = 0; int aktuellerZustand = 0; // 0=Rot, 1=Rot+Gelb, 2=Grün, 3=Gelb void setup() { pinMode(ROT, OUTPUT); pinMode(GELB, OUTPUT); pinMode(GRUEN, OUTPUT); // Startzustand: Rot digitalWrite(ROT, HIGH); letzteSchaltung = millis(); } void loop() { unsigned long aktuelleZeit = millis(); switch(aktuellerZustand) { case 0: // Rot if(aktuelleZeit - letzteSchaltung >= ROT_ZEIT) { digitalWrite(GELB, HIGH); // Rot + Gelb aktuellerZustand = 1; letzteSchaltung = aktuelleZeit; } break; case 1: // Rot + Gelb if(aktuelleZeit - letzteSchaltung >= 1000) { digitalWrite(ROT, LOW); digitalWrite(GELB, LOW); digitalWrite(GRUEN, HIGH); // Grün aktuellerZustand = 2; letzteSchaltung = aktuelleZeit; } break; case 2: // Grün if(aktuelleZeit - letzteSchaltung >= GRUEN_ZEIT) { digitalWrite(GRUEN, LOW); digitalWrite(GELB, HIGH); // Gelb aktuellerZustand = 3; letzteSchaltung = aktuelleZeit; } break; case 3: // Gelb if(aktuelleZeit - letzteSchaltung >= GELB_ZEIT) { digitalWrite(GELB, LOW); digitalWrite(ROT, HIGH); // Rot aktuellerZustand = 0; letzteSchaltung = aktuelleZeit; } break; } }

📡 Beispiel 3: I²C Kommunikation mit Sensor

Arduino Code - I²C Temperatursensor LM75
#include // I²C-Adresse des LM75 Temperatursensors const int LM75_ADRESSE = 0x48; void setup() { Serial.begin(9600); // Serielle Kommunikation starten Wire.begin(); // I²C initialisieren Serial.println("LM75 Temperatursensor Test"); } void loop() { // Temperatur-Register lesen (Register 0x00) Wire.beginTransmission(LM75_ADRESSE); Wire.write(0x00); // Register-Adresse senden Wire.endTransmission(); // 2 Bytes vom Sensor lesen Wire.requestFrom(LM75_ADRESSE, 2); if(Wire.available() >= 2) { // Erste Byte: Hauptteil der Temperatur int highByte = Wire.read(); // Zweite Byte: Nachkommastellen int lowByte = Wire.read(); // Temperatur berechnen (LM75 spezifisch) float temperatur = highByte; if(lowByte & 0x80) temperatur += 0.5; // 0.5°C Auflösung // Ergebnis ausgeben Serial.print("Temperatur: "); Serial.print(temperatur); Serial.println(" °C"); } delay(2000); // 2 Sekunden warten }

⚡ Beispiel 4: PWM für Motorsteuerung

Arduino Code - DC Motor mit PWM
// Motor-Pins const int MOTOR_PWM = 9; // PWM-Pin für Geschwindigkeit const int MOTOR_DIR1 = 7; // Richtungssteuerung 1 const int MOTOR_DIR2 = 8; // Richtungssteuerung 2 // Potentiometer für Geschwindigkeitssteuerung const int POTI_PIN = A0; void setup() { pinMode(MOTOR_PWM, OUTPUT); pinMode(MOTOR_DIR1, OUTPUT); pinMode(MOTOR_DIR2, OUTPUT); Serial.begin(9600); } void loop() { // Potentiometer lesen (0-1023) int potiWert = analogRead(POTI_PIN); // In Geschwindigkeit umrechnen (-255 bis +255) int geschwindigkeit = map(potiWert, 0, 1023, -255, 255); // Motorrichtung und -geschwindigkeit einstellen if(geschwindigkeit > 0) { // Vorwärts digitalWrite(MOTOR_DIR1, HIGH); digitalWrite(MOTOR_DIR2, LOW); analogWrite(MOTOR_PWM, geschwindigkeit); Serial.print("Vorwärts: "); Serial.println(geschwindigkeit); } else if(geschwindigkeit < 0) { // Rückwärts digitalWrite(MOTOR_DIR1, LOW); digitalWrite(MOTOR_DIR2, HIGH); analogWrite(MOTOR_PWM, -geschwindigkeit); // Betrag verwenden Serial.print("Rückwärts: "); Serial.println(-geschwindigkeit); } else { // Stop digitalWrite(MOTOR_DIR1, LOW); digitalWrite(MOTOR_DIR2, LOW); analogWrite(MOTOR_PWM, 0); Serial.println("Motor gestoppt"); } delay(100); // Kurze Pause }

🚨 Häufige Fehler und Lösungen

⚠️ Typische Anfängerfehler

  • Offene Eingänge: Digitale Eingänge ohne Pull-Up-Widerstand "schweben" und können zufällige Werte haben
  • Stromüberlastung: Ein GPIO-Pin kann max. 20-40mA liefern - für LEDs Vorwiderstände verwenden!
  • Falsche Referenzspannung: Bei ADC-Messungen die richtige Referenz (meist VCC) verwenden
  • Timing-Probleme: delay() blockiert den ganzen Mikrocontroller - besser Timer verwenden
  • Stack Overflow: Zu tiefe Rekursion oder zu große lokale Variablen

💡 Best Practices

  • Konstanten definieren: Verwende #define für Pin-Nummern und Werte
  • Funktionen verwenden: Teile Code in kleine, wiederverwendbare Funktionen
  • Kommentare schreiben: Erkläre, was dein Code macht
  • Hardware testen: Teste neue Hardware erst mit einfachen Programmen
  • Schaltplan zeichnen: Dokumentiere deine Verschaltung

📋 Zusammenfassung

🎯 Das Wichtigste im Überblick

Ein Mikrocontroller ist ein vollständiger Computer auf einem Chip, der speziell für eingebettete Anwendungen entwickelt wurde:

🏗️ Hauptkomponenten

  • CPU: Führt Programme aus (ALU, Steuerwerk, Register)
  • Speicher: RAM (flüchtig), ROM/Flash (dauerhaft), EEPROM (konfigurierbar)
  • Peripherie: GPIO, Timer, Kommunikationsschnittstellen, Wandler
  • Bus-System: Verbindet alle Komponenten miteinander

🏛️ Architekturen

  • Harvard: Getrennte Speicher für Code und Daten (schneller)
  • Von-Neumann: Gemeinsamer Speicher (flexibler)
  • RISC: Einfache Befehle, Load-Store-Architektur
  • CISC: Komplexe Befehle, direkter Speicherzugriff

💻 Programmierung

  • Sprachen: Assembly, C, C++, Arduino IDE
  • Werkzeuge: IDE, Compiler, Debugger, Programmer
  • Konzepte: Register, Interrupts, Timer, Kommunikation

🎓 Prüfungsrelevante Themen

  • Unterschied CPU vs. Mikrocontroller
  • Harvard vs. Von-Neumann Architektur
  • RISC vs. CISC Prinzipien
  • Speicherarten (RAM, ROM, Flash, EEPROM)
  • Register-System (ARM Cortex-M)
  • GPIO-Programmierung
  • PWM-Funktionsweise
  • Kommunikationsschnittstellen (UART, I²C, SPI)
  • A/D und D/A Wandlung
