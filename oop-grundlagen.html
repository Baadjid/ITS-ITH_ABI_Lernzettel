📖 Was ist Objektorientierte Programmierung?

Grundlagen und Motivation für OOP

Die Objektorientierte Programmierung (OOP) ist ein Programmierparadigma, das die Softwareentwicklung revolutioniert hat. Statt Funktionen und Daten getrennt zu betrachten, organisiert OOP Code in Objekten, die sowohl Daten (Attribute) als auch Verhalten (Methoden) kombinieren.

🏠 Analogie: Ein Haus bauen

Stell dir vor, du möchtest ein Haus bauen:

  • Ohne OOP: Du würdest jeden Raum einzeln planen, ohne Struktur oder Wiederverwendung
  • Mit OOP: Du erstellst Baupläne (Klassen) für verschiedene Räume und baust dann konkrete Räume (Objekte) nach diesen Plänen
💡 Warum OOP verwenden?
  • Code-Wiederverwendung durch Vererbung
  • Bessere Organisation großer Programme
  • Wartbarkeit und Erweiterbarkeit
  • Nähe zur realen Welt (natürlichere Modellierung)
  • Teamarbeit durch klare Schnittstellen

🔒 Prinzip 1: Kapselung (Encapsulation)

Daten und Methoden zusammenfassen und vor direktem Zugriff schützen
🔐
Kapselung
Verstecke interne Details, zeige nur notwendige Schnittstellen

Kapselung bedeutet, dass die interne Struktur eines Objekts von außen nicht direkt zugänglich ist. Stattdessen werden kontrollierte Zugriffsmethoden bereitgestellt.

🚗 Analogie: Auto fahren

Wenn du Auto fährst, musst du nicht wissen, wie der Motor funktioniert. Du nutzt nur das Lenkrad, die Pedale und das Armaturenbrett. Die komplexe Technik ist "gekapselt" und vor dir verborgen.

Umsetzung in Java:

💻 Beispiel: Auto-Klasse mit Kapselung Klicken zum Kopieren 
public class Auto {
    // Private Attribute - von außen nicht direkt zugreifbar
    private String marke;
    private int geschwindigkeit;
    private boolean motorLaeuft;
    
    // Konstruktor
    public Auto(String marke) {
        this.marke = marke;
        this.geschwindigkeit = 0;
        this.motorLaeuft = false;
    }
    
    // Öffentliche Methoden - kontrollierter Zugriff
    public void starten() {
        if (!motorLaeuft) {
            motorLaeuft = true;
            System.out.println("Motor gestartet!");
        }
    }
    
    public void beschleunigen(int kmh) {
        if (motorLaeuft && kmh > 0) {
            geschwindigkeit += kmh;
            if (geschwindigkeit > 200) {
                geschwindigkeit = 200; // Geschwindigkeitsbegrenzung
            }
        }
    }
    
    // Getter-Methoden für kontrollierten Lesezugriff
    public int getGeschwindigkeit() {
        return geschwindigkeit;
    }
    
    public String getMarke() {
        return marke;
    }
}
🔑 Sichtbarkeitsmodifikatoren in Java
Modifikator Zugriff Verwendung
private Nur innerhalb der Klasse Interne Daten und Hilfsmethoden
protected Klasse + Unterklassen + Package Vererbungshierarchien
public Von überall Öffentliche Schnittstellen
(package-private) Nur innerhalb des Packages Interne Klassen eines Pakets

🌳 Prinzip 2: Vererbung (Inheritance)

Code wiederverwenden durch Klassenhierarchien
🧬
Vererbung
Neue Klassen basierend auf existierenden Klassen erstellen

Vererbung ermöglicht es, neue Klassen zu erstellen, die Eigenschaften und Methoden bereits existierender Klassen übernehmen und erweitern.

👨‍👩‍👧‍👦 Analogie: Familie

Wie in einer Familie Kinder Eigenschaften von ihren Eltern erben (Augenfarbe, Größe), erben Unterklassen Attribute und Methoden von ihrer Oberklasse - können aber auch neue hinzufügen.

Umsetzung in Java:

💻 Beispiel: Fahrzeug-Hierarchie Klicken zum Kopieren 
// Oberklasse (Superklasse)
public class Fahrzeug {
    protected String marke;
    protected int geschwindigkeit;
    
    public Fahrzeug(String marke) {
        this.marke = marke;
        this.geschwindigkeit = 0;
    }
    
    public void beschleunigen(int kmh) {
        geschwindigkeit += kmh;
        System.out.println("Beschleunige auf " + geschwindigkeit + " km/h");
    }
    
    public void bremsen() {
        geschwindigkeit = 0;
        System.out.println("Fahrzeug steht");
    }
}

// Unterklasse (Subklasse) - erbt von Fahrzeug
public class Auto extends Fahrzeug {
    private int anzahlTueren;
    
    public Auto(String marke, int tueren) {
        super(marke); // Aufruf des Oberklassen-Konstruktors
        this.anzahlTueren = tueren;
    }
    
    // Eigene Methode, die nur Auto hat
    public void hupen() {
        System.out.println("BEEP BEEP!");
    }
    
    // Überschreibung einer geerbten Methode
    @Override
    public void beschleunigen(int kmh) {
        super.beschleunigen(kmh); // Rufe Oberklassen-Methode auf
        if (geschwindigkeit > 130) {
            System.out.println("Achtung: Geschwindigkeitsbegrenzung!");
        }
    }
}

// Weitere Unterklasse
public class Motorrad extends Fahrzeug {
    private boolean hatBeiwagen;
    
    public Motorrad(String marke, boolean beiwagen) {
        super(marke);
        this.hatBeiwagen = beiwagen;
    }
    
    public void wheelie() {
        if (geschwindigkeit > 50) {
            System.out.println("Wheelie!");
        }
    }
}
🔄 Wichtige Vererbungskonzepte
  • super(): Aufruf des Oberklassen-Konstruktors
  • @Override: Kennzeichnet überschriebene Methoden
  • protected: Zugriff für Unterklassen ermöglichen
  • final: Verhindert Vererbung/Überschreibung

🎭 Prinzip 3: Polymorphie (Polymorphism)

Ein Interface, verschiedene Implementierungen
🎪
Polymorphie
Gleiche Schnittstelle, unterschiedliches Verhalten

Polymorphie ermöglicht es, dass Objekte verschiedener Klassen über die gleiche Schnittstelle angesprochen werden können, aber unterschiedlich reagieren.

🎵 Analogie: Musikinstrumente

Alle Musikinstrumente haben die Methode "spielen()", aber jedes Instrument klingt anders: Eine Gitarre zupft, ein Klavier hämmert Tasten, eine Flöte bläst Luft. Gleiche Aktion, unterschiedliches Ergebnis!

Umsetzung in Java:

💻 Beispiel: Polymorphie mit Tieren Klicken zum Kopieren 
// Oberklasse
abstract class Tier {
    protected String name;
    
    public Tier(String name) {
        this.name = name;
    }
    
    // Abstrakte Methode - muss von Unterklassen implementiert werden
    public abstract void machLaut();
    
    // Normale Methode
    public void schlafen() {
        System.out.println(name + " schläft...");
    }
}

// Verschiedene Unterklassen mit eigenem Verhalten
class Hund extends Tier {
    public Hund(String name) {
        super(name);
    }
    
    @Override
    public void machLaut() {
        System.out.println(name + " bellt: Wuff Wuff!");
    }
}

class Katze extends Tier {
    public Katze(String name) {
        super(name);
    }
    
    @Override
    public void machLaut() {
        System.out.println(name + " miaut: Miau!");
    }
}

class Kuh extends Tier {
    public Kuh(String name) {
        super(name);
    }
    
    @Override
    public void machLaut() {
        System.out.println(name + " macht: Muh!");
    }
}

// Polymorphie in Aktion
public class TierTest {
    public static void main(String[] args) {
        // Array mit verschiedenen Tieren
        Tier[] tiere = {
            new Hund("Bello"),
            new Katze("Whiskers"),
            new Kuh("Liesel")
        };
        
        // Polymorphie: Gleicher Aufruf, verschiedenes Verhalten
        for (Tier tier : tiere) {
            tier.machLaut(); // Jedes Tier reagiert anders!
            tier.schlafen();  // Gemeinsames Verhalten
        }
    }
}

🎨 Prinzip 4: Abstraktion (Abstraction)

Komplexität verbergen, Wesentliches hervorheben
🧩
Abstraktion
Fokus auf das WAS, nicht auf das WIE

Abstraktion bedeutet, komplexe Implementierungsdetails zu verbergen und nur die wesentlichen Funktionalitäten nach außen zu zeigen.

📱 Analogie: Smartphone

Du nutzt Apps auf deinem Smartphone, ohne zu wissen, wie der Prozessor funktioniert oder wie die Daten gespeichert werden. Die komplexe Technik ist abstrahiert - du siehst nur die einfache Benutzeroberfläche.

Umsetzung in Java:

💻 Beispiel: Abstrakte Klasse vs. Interface Klicken zum Kopieren 
// Abstrakte Klasse - kann konkrete und abstrakte Methoden haben
abstract class Form {
    protected String farbe;
    
    public Form(String farbe) {
        this.farbe = farbe;
    }
    
    // Konkrete Methode
    public void setFarbe(String farbe) {
        this.farbe = farbe;
    }
    
    // Abstrakte Methoden - müssen implementiert werden
    public abstract double berechneFlaeche();
    public abstract double berechneUmfang();
}

// Interface - nur abstrakte Methoden (bis Java 8)
interface Zeichenbar {
    void zeichnen();
    void bewegen(int x, int y);
}

// Implementierung
class Kreis extends Form implements Zeichenbar {
    private double radius;
    
    public Kreis(String farbe, double radius) {
        super(farbe);
        this.radius = radius;
    }
    
    @Override
    public double berechneFlaeche() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public double berechneUmfang() {
        return 2 * Math.PI * radius;
    }
    
    @Override
    public void zeichnen() {
        System.out.println("Zeichne einen " + farbe + "en Kreis mit Radius " + radius);
    }
    
    @Override
    public void bewegen(int x, int y) {
        System.out.println("Bewege Kreis zu Position (" + x + ", " + y + ")");
    }
}

class Rechteck extends Form implements Zeichenbar {
    private double breite;
    private double hoehe;
    
    public Rechteck(String farbe, double breite, double hoehe) {
        super(farbe);
        this.breite = breite;
        this.hoehe = hoehe;
    }
    
    @Override
    public double berechneFlaeche() {
        return breite * hoehe;
    }
    
    @Override
    public double berechneUmfang() {
        return 2 * (breite + hoehe);
    }
    
    @Override
    public void zeichnen() {
        System.out.println("Zeichne ein " + farbe + "es Rechteck " + breite + "x" + hoehe);
    }
    
    @Override
    public void bewegen(int x, int y) {
        System.out.println("Bewege Rechteck zu Position (" + x + ", " + y + ")");
    }
}

// Verwendung der Abstraktion
public class FormenDemo {
    public static void main(String[] args) {
        // Polymorphie mit abstrakten Klassen und Interfaces
        Form[] formen = {
            new Kreis("rot", 5.0),
            new Rechteck("blau", 4.0, 6.0)
        };
        
        for (Form form : formen) {
            form.zeichnen(); // Interface-Methode
            System.out.println("Fläche: " + form.berechneFlaeche()); // Abstrakte Methode
            form.bewegen(10, 20); // Interface-Methode
            System.out.println("---");
        }
    }
}
🤔 Abstrakte Klasse vs. Interface
Aspekt Abstrakte Klasse Interface
Methodenarten Konkrete + abstrakte Methoden Nur abstrakte Methoden (bis Java 8)
Attribute Alle Arten von Attributen Nur public static final
Konstruktor Ja Nein
Mehrfachvererbung Nein (extends nur eine) Ja (implements mehrere)
Verwendung Gemeinsame Basis mit Code Vertrag/Kontrakt definieren

🏛️ Die vier Säulen der OOP - Zusammenfassung

Alle Prinzipien im Überblick
🎯 Merksatz für die Prüfung

"KAVA" - Kapselung, Abstraktion, Vererbung, polymAmorphie

Prinzip Definition Java-Schlüsselwort Zweck
🔒 Kapselung Daten und Methoden zusammenfassen, Zugriff kontrollieren private, protected, public Datenintegrität, Sicherheit
🌳 Vererbung Eigenschaften und Methoden von anderen Klassen übernehmen extends, super Code-Wiederverwendung
🎭 Polymorphie Gleiche Schnittstelle, verschiedenes Verhalten @Override, abstract Flexibilität, Erweiterbarkeit
🎨 Abstraktion Komplexität verbergen, Wesentliches zeigen abstract, interface Vereinfachung, Struktur
🔧 Alle Prinzipien in einem Beispiel Klicken zum Kopieren 
// Abstraktion durch Interface
interface Fahrzeugfunktionen {
    void starten();
    void stoppen();
    void beschleunigen(int kmh);
}

// Abstrakte Basisklasse mit gemeinsamen Eigenschaften
abstract class Fahrzeug implements Fahrzeugfunktionen {
    // Kapselung: private Attribute
    private String marke;
    private int geschwindigkeit;
    protected boolean istGestartet; // protected für Unterklassen
    
    // Konstruktor
    public Fahrzeug(String marke) {
        this.marke = marke;
        this.geschwindigkeit = 0;
        this.istGestartet = false;
    }
    
    // Kapselung: Kontrollierter Zugriff durch Getter
    public String getMarke() { return marke; }
    public int getGeschwindigkeit() { return geschwindigkeit; }
    
    // Gemeinsame Implementierung
    public void starten() {
        istGestartet = true;
        System.out.println(marke + " wurde gestartet");
    }
    
    public void stoppen() {
        istGestartet = false;
        geschwindigkeit = 0;
        System.out.println(marke + " wurde gestoppt");
    }
    
    // Kapselung: Protected Methode für Unterklassen
    protected void setGeschwindigkeit(int kmh) {
        if (kmh >= 0 && kmh <= getMaxGeschwindigkeit()) {
            this.geschwindigkeit = kmh;
        }
    }
    
    // Abstraktion: Muss von Unterklassen implementiert werden
    public abstract int getMaxGeschwindigkeit();
    public abstract String getFahrzeugtyp();
}

// Vererbung: Auto erbt von Fahrzeug
class Auto extends Fahrzeug {
    private int anzahlTueren;
    
    public Auto(String marke, int tueren) {
        super(marke); // Aufruf Oberklassen-Konstruktor
        this.anzahlTueren = tueren;
    }
    
    // Polymorphie: Eigene Implementierung der abstrakten Methode
    @Override
    public void beschleunigen(int kmh) {
        if (istGestartet) {
            setGeschwindigkeit(getGeschwindigkeit() + kmh);
            System.out.println("Auto beschleunigt auf " + getGeschwindigkeit() + " km/h");
        }
    }
    
    @Override
    public int getMaxGeschwindigkeit() { return 200; }
    
    @Override
    public String getFahrzeugtyp() { return "PKW"; }
    
    // Spezifische Auto-Methode
    public void hupen() {
        System.out.println("BEEP BEEP!");
    }
}

// Vererbung: Motorrad erbt von Fahrzeug
class Motorrad extends Fahrzeug {
    public Motorrad(String marke) {
        super(marke);
    }
    
    // Polymorphie: Andere Implementierung
    @Override
    public void beschleunigen(int kmh) {
        if (istGestartet) {
            setGeschwindigkeit(getGeschwindigkeit() + kmh);
            System.out.println("Motorrad beschleunigt rasant auf " + getGeschwindigkeit() + " km/h");
        }
    }
    
    @Override
    public int getMaxGeschwindigkeit() { return 300; }
    
    @Override
    public String getFahrzeugtyp() { return "Zweirad"; }
}

// Polymorphie in der Anwendung
public class FahrzeugDemo {
    public static void main(String[] args) {
        // Polymorphie: Verschiedene Objekte, gleiche Referenz
        Fahrzeug[] fuhrpark = {
            new Auto("BMW", 4),
            new Motorrad("Harley Davidson")
        };
        
        // Polymorphie: Gleiche Methoden, verschiedenes Verhalten
        for (Fahrzeug fahrzeug : fuhrpark) {
            fahrzeug.starten();
            fahrzeug.beschleunigen(50);
            System.out.println("Typ: " + fahrzeug.getFahrzeugtyp());
            fahrzeug.stoppen();
            System.out.println("---");
        }
    }
}

⚠️ Häufige Fehler und Fallstricke

Typische Probleme bei der OOP-Anwendung vermeiden
🚫 Häufige Fehlerquellen
  • Falsche Sichtbarkeit: Alles public machen statt gezielter Kapselung
  • Vererbung missbrauchen: "Hat-eine"-Beziehung statt "Ist-eine"-Beziehung
  • Vergessenes @Override: Methoden überladen statt überschreiben
  • super() vergessen: Oberklassen-Konstruktor nicht aufrufen
  • NullPointerException: Fehlende Null-Checks bei Objektreferenzen
❌ Falsch vs. ✅ Richtig Klicken zum Kopieren 
// ❌ FALSCH: Keine Kapselung
class SchlechteKlasse {
    public String name;  // Direkter Zugriff möglich
    public int alter;    // Keine Validierung möglich
}

// ✅ RICHTIG: Proper Kapselung
class GuteKlasse {
    private String name;
    private int alter;
    
    public void setAlter(int alter) {
        if (alter >= 0 && alter <= 120) {  // Validierung!
            this.alter = alter;
        } else {
            throw new IllegalArgumentException("Ungültiges Alter: " + alter);
        }
    }
    
    public int getAlter() { return alter; }
}

// ❌ FALSCH: Falsche Vererbung (Komposition wäre besser)
class Auto extends Motor {  // Auto IST NICHT ein Motor!
}

// ✅ RICHTIG: Komposition verwenden
class Auto {
    private Motor motor;  // Auto HAT einen Motor
    
    public Auto() {
        this.motor = new Motor();
    }
}

// ❌ FALSCH: Überladen statt Überschreiben
class Hund extends Tier {
    public void machLaut(String laut) {  // Andere Signatur = Überladen!
        System.out.println(laut);
    }
}

// ✅ RICHTIG: Überschreiben mit @Override
class Hund extends Tier {
    @Override
    public void machLaut() {  // Gleiche Signatur = Überschreiben!
        System.out.println("Wuff!");
    }
}

📋 Übungsaufgabe: Bibliothekssystem Klicken zum Kopieren 
/**
 * Übung: Erstelle ein Bibliothekssystem mit folgenden Anforderungen:
 * 1. Abstrakte Basisklasse "Medium" mit Titel, Autor, Jahr
 * 2. Unterklassen "Buch" und "DVD" mit spezifischen Eigenschaften
 * 3. Interface "Ausleihbar" mit Methoden ausleihen() und zurückgeben()
 * 4. Alle OOP-Prinzipien anwenden
 */

// Lösung:
abstract class Medium {
    private String titel;
    private String autor;
    private int erscheinungsjahr;
    protected boolean istAusgeliehen;
    
    public Medium(String titel, String autor, int jahr) {
        this.titel = titel;
        this.autor = autor;
        this.erscheinungsjahr = jahr;
        this.istAusgeliehen = false;
    }
    
    // Getter-Methoden (Kapselung)
    public String getTitel() { return titel; }
    public String getAutor() { return autor; }
    public int getErscheinungsjahr() { return erscheinungsjahr; }
    
    // Abstrakte Methode (Abstraktion)
    public abstract String getMedientyp();
    
    // Template-Method Pattern
    public void zeigeMedieninfo() {
        System.out.println(getMedientyp() + ": " + titel + " von " + autor + " (" + erscheinungsjahr + ")");
    }
}

interface Ausleihbar {
    boolean ausleihen();
    boolean zurückgeben();
    boolean istVerfuegbar();
}

class Buch extends Medium implements Ausleihbar {
    private int seitenzahl;
    private String isbn;
    
    public Buch(String titel, String autor, int jahr, int seiten, String isbn) {
        super(titel, autor, jahr);
        this.seitenzahl = seiten;
        this.isbn = isbn;
    }
    
    @Override
    public String getMedientyp() {
        return "Buch";
    }
    
    @Override
    public boolean ausleihen() {
        if (!istAusgeliehen) {
            istAusgeliehen = true;
            System.out.println("Buch '" + getTitel() + "' wurde ausgeliehen");
            return true;
        }
        return false;
    }
    
    @Override
    public boolean zurückgeben() {
        if (istAusgeliehen) {
            istAusgeliehen = false;
            System.out.println("Buch '" + getTitel() + "' wurde zurückgegeben");
            return true;
        }
        return false;
    }
    
    @Override
    public boolean istVerfuegbar() {
        return !istAusgeliehen;
    }
}

class DVD extends Medium implements Ausleihbar {
    private int laufzeitMinuten;
    private String genre;
    
    public DVD(String titel, String regisseur, int jahr, int laufzeit, String genre) {
        super(titel, regisseur, jahr);
        this.laufzeitMinuten = laufzeit;
        this.genre = genre;
    }
    
    @Override
    public String getMedientyp() {
        return "DVD";
    }
    
    @Override
    public boolean ausleihen() {
        if (!istAusgeliehen) {
            istAusgeliehen = true;
            System.out.println("DVD '" + getTitel() + "' wurde ausgeliehen");
            return true;
        }
        return false;
    }
    
    @Override
    public boolean zurückgeben() {
        if (istAusgeliehen) {
            istAusgeliehen = false;
            System.out.println("DVD '" + getTitel() + "' wurde zurückgegeben");
            return true;
        }
        return false;
    }
    
    @Override
    public boolean istVerfuegbar() {
        return !istAusgeliehen;
    }
}
