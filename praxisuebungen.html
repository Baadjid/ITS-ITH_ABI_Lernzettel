Übung 1.1: Logikgatter verstehen und anwenden

Grundlagen

💡 Was sind Logikgatter?

Logikgatter sind die Grundbausteine der Digitaltechnik. Sie verarbeiten binäre Eingangssignale (0 oder 1) und erzeugen ein binäres Ausgangssignal nach einer bestimmten logischen Regel.

Warum sind sie wichtig? Alle digitalen Geräte - von Taschenrechnern bis zu Smartphones - basieren auf Millionen von Logikgattern, die zusammenarbeiten.

🎯 Aufgabe: Grundlegende Gatter kennenlernen

AND-Gatter (UND-Gatter):

0 1 0 Ein Eingang ist AUS → Ausgang ist AUS
1 0 0 Ein Eingang ist AUS → Ausgang ist AUS
1 1 1 Beide Eingänge sind AN → Ausgang ist AN

🔍 Schritt-für-Schritt Analyse:

Übung 1.2: RS-FlipFlop verstehen

Mittelstufe

💡 Was ist ein FlipFlop?

Ein FlipFlop ist ein digitaler Speicher, der einen Bit (0 oder 1) dauerhaft speichern kann, bis er überschrieben wird.

RS-FlipFlop hat zwei Eingänge:

  • S (Set): Setzt den Ausgang Q auf 1
  • R (Reset): Setzt den Ausgang Q auf 0

🎮 RS-FlipFlop Simulator:

Testen Sie verschiedene Eingangskombinationen:

Q = gespeicherter Wert

Aktueller Zustand wird zwischen den Operationen gespeichert!

🔍 RS-FlipFlop Schritt für Schritt:

  1. R=0, S=0: Nichts passiert, der alte Wert bleibt gespeichert
  2. R=0, S=1: SET - Der Ausgang Q wird auf 1 gesetzt
  3. R=1, S=0: RESET - Der Ausgang Q wird auf 0 zurückgesetzt
  4. R=1, S=1: VERBOTEN - Führt zu undefined Zustand!

💡 Warum Java?

Java ist eine der wichtigsten Programmiersprachen:

  • Plattformunabhängig: "Write once, run anywhere"
  • Objektorientiert: Strukturiert große Programme
  • Typsicher: Verhindert viele Programmierfehler

Übung 2.2: Verzweigungen und Schleifen

Mittelstufe

💡 Kontrollstrukturen steuern den Programmablauf

Verzweigungen (if-else): Programm trifft Entscheidungen

Schleifen: Code wird wiederholt ausgeführt

🎯 Aufgabe: Zahlenraten mit verschiedenen Schwierigkeitsgraden

import java.util.Scanner; import java.util.Random; public class Zahlenraten { public static void main(String[] args) { Scanner scanner = new Scanner(System.in); Random random = new Random(); System.out.println("=== ZAHLENRATEN ==="); System.out.println("1 = Einfach (1-10)"); System.out.println("2 = Mittel (1-50)"); System.out.println("3 = Schwer (1-100)"); int schwierigkeit = scanner.nextInt(); int maxZahl, maxVersuche; switch (schwierigkeit) { case 1: maxZahl = 10; maxVersuche = 5; break; case 2: maxZahl = 50; maxVersuche = 7; break; case 3: maxZahl = 100; maxVersuche = 10; break; default: maxZahl = 10; maxVersuche = 5; } int zufallsZahl = random.nextInt(maxZahl) + 1; int versuche = 0; boolean erraten = false; while (versuche < maxVersuche && !erraten) { versuche++; System.out.print("Versuch " + versuche + ": "); int tipp = scanner.nextInt(); if (tipp == zufallsZahl) { erraten = true; System.out.println("🎉 Richtig!"); } else if (tipp < zufallsZahl) { System.out.println("📈 Zu niedrig!"); } else { System.out.println("📉 Zu hoch!"); } } if (!erraten) { System.out.println("💔 Die Zahl war: " + zufallsZahl); } scanner.close(); } }

🔍 Kontrollstrukturen im Detail:

  1. switch-case: Mehrfachverzweigung für sauberen Code
  2. while-Schleife: Wiederholt Code solange Bedingung wahr ist
  3. if-else-if: Mehrere Bedingungen nacheinander prüfen
  4. boolean-Variablen: Steuern den Programmfluss

🗄️ 3. SQL und Datenbanken

Level: Mittelstufe bis Fortgeschritten | Zeitaufwand: 5-7 Stunden

Übung 3.1: Erste SQL-Abfragen

Grundlagen

💡 Was sind relationale Datenbanken?

Datenbanken speichern große Mengen strukturierter Daten effizient.

  • Tabellen: Sammlung verwandter Daten
  • SQL: Structured Query Language
  • Relationen: Beziehungen zwischen Tabellen

🎯 Beispiel-Datenbank: Schulverwaltung

-- Tabelle erstellen CREATE TABLE schueler ( s_id INT PRIMARY KEY AUTO_INCREMENT, vorname VARCHAR(50) NOT NULL, nachname VARCHAR(50) NOT NULL, geburtsdatum DATE, klasse_id INT ); -- Daten einfügen INSERT INTO schueler VALUES (1, 'Anna', 'Schmidt', '2007-03-15', 1), (2, 'Max', 'Müller', '2006-08-22', 1), (3, 'Lisa', 'Weber', '2007-01-10', 2);

🎯 Grundlegende SELECT-Abfragen

-- Alle Schüler anzeigen SELECT * FROM schueler; -- Nur Namen anzeigen SELECT vorname, nachname FROM schueler; -- Mit Bedingung filtern SELECT * FROM schueler WHERE klasse_id = 1; -- Sortiert ausgeben SELECT vorname, nachname FROM schueler ORDER BY nachname;

Übung 3.2: JOIN-Operationen

Mittelstufe

💡 Was sind JOINs?

JOINs verknüpfen Daten aus mehreren Tabellen. Sie sind der Kern relationaler Datenbanken.

🎯 INNER JOIN - Schüler mit Klassendaten

-- Schüler mit Klasseninformationen SELECT s.vorname, s.nachname, k.klassenname FROM schueler s INNER JOIN klassen k ON s.klasse_id = k.k_id ORDER BY k.klassenname, s.nachname; -- Notendurchschnitt pro Schüler SELECT s.vorname, s.nachname, ROUND(AVG(n.note), 2) as durchschnitt FROM schueler s INNER JOIN noten n ON s.s_id = n.schueler_id GROUP BY s.s_id, s.vorname, s.nachname ORDER BY durchschnitt;
