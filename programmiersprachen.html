🔧 Was ist Software?

📖 Definition: Software

Software sind alle Programme, die auf einem Computer ausgeführt werden können. Sie bestehen aus Anweisungen, die dem Computer genau sagen, was er tun soll.

Im Gegensatz zur Hardware (die physischen Teile des Computers) ist Software nicht greifbar - sie existiert nur als Code und Daten.

🎯 Warum gibt es so viele Programmiersprachen?

Es gibt hunderte verschiedene Programmiersprachen, weil jede für bestimmte Zwecke optimiert ist:

📊 Die Hierarchie der Programmiersprachen

💡 Grundprinzip

Computer verstehen nur Maschinensprache (Nullen und Einsen). Alle höheren Programmiersprachen müssen zuerst in Maschinensprache übersetzt werden, bevor der Computer sie ausführen kann.

🔄 Interpreter vs. Compiler

⚠️ Wichtig zu verstehen

Höhere Programmiersprachen müssen in Maschinensprache übersetzt werden. Dafür gibt es zwei verschiedene Methoden: Interpreter und Compiler.

🔄 Interpreter

Funktionsweise: Übersetzt während das Programm läuft, Zeile für Zeile

Vorteil: Programm kann sofort gestartet werden

Nachteil: Langsam, da jede Zeile bei jedem Durchlauf neu übersetzt wird

// Beispiele für interpretierte Sprachen: - JavaScript - PHP - Python - Basic

⚙️ Compiler

Funktionsweise: Übersetzt das komplette Programm vor der Ausführung

Vorteil: Sehr schnell bei der Ausführung

Nachteil: Kompilieren kostet Zeit, Syntaxfehler werden erst beim Kompilieren erkannt

// Beispiele für kompilierte Sprachen: - Java - C/C++ - C# - Go

🔧 Der Übersetzungsvorgang im Detail

🔍 Beispiel: Interpreter

1. Programmierer schreibt: print("Hallo Welt") 2. Interpreter liest diese Zeile 3. Interpreter übersetzt sie in Maschinencode 4. Computer führt die Zeile aus 5. Interpreter geht zur nächsten Zeile # Bei jedem Programmstart passiert das wieder!

🔍 Beispiel: Compiler

1. Programmierer schreibt: System.out.println("Hallo Welt"); 2. Compiler übersetzt das komplette Programm 3. Erstellt eine ausführbare Datei (.exe, .class) 4. Computer kann die Datei direkt ausführen // Kompilieren muss nur einmal gemacht werden!

🔧 Die 3 Grundbausteine jedes Programms

⚠️ Fundamentales Konzept

JEDES Programm besteht aus nur drei Grundbausteinen! Wenn du diese verstehst, kannst du jedes Programm verstehen.

1️⃣ Sequenz (Reihenfolge)

📖 Was ist eine Sequenz?

Befehle werden nacheinander, in der richtigen Reihenfolge ausgeführt. Wie beim Anziehen: erst Socken, dann Schuhe!

🔍 Beispiel: Sequenz

// Diese Befehle werden von oben nach unten ausgeführt int a = 5; // 1. Variable a wird auf 5 gesetzt int b = 3; // 2. Variable b wird auf 3 gesetzt int summe = a + b; // 3. summe wird berechnet (5 + 3 = 8) System.out.println(summe); // 4. Das Ergebnis wird ausgegeben

2️⃣ Auswahl (Verzweigung)

📖 Was ist eine Auswahl?

Das Programm entscheidet basierend auf einer Bedingung, welcher Code ausgeführt wird. Wie bei einer Kreuzung: links oder rechts?

🔍 Beispiel: Auswahl (if-else)

int alter = 16; if (alter >= 18) { System.out.println("Du darfst wählen!"); // Wird NUR ausgeführt wenn alter >= 18 } else { System.out.println("Du darfst noch nicht wählen."); // Wird ausgeführt wenn alter < 18 } // Da alter = 16, wird "Du darfst noch nicht wählen." ausgegeben

3️⃣ Wiederholung (Schleife)

📖 Was ist eine Wiederholung?

Ein Code-Block wird mehrfach ausgeführt, solange eine Bedingung erfüllt ist. Wie beim Zähneputzen: so lange schrubben, bis alle Zähne sauber sind!

🔍 Beispiel: Schleife (while)

int countdown = 5; while (countdown > 0) { // Solange countdown größer als 0 System.out.println("Noch " + countdown + " Sekunden"); countdown = countdown - 1; // countdown verringern } System.out.println("Zeit abgelaufen!"); /* Ausgabe: * Noch 5 Sekunden * Noch 4 Sekunden * Noch 3 Sekunden * Noch 2 Sekunden * Noch 1 Sekunden * Zeit abgelaufen! */

🎯 Kombinationen der Grundbausteine

  • Komplexe Programme sind nur Kombinationen dieser drei Bausteine
  • Schleifen können Verzweigungen enthalten
  • Verzweigungen können weitere Schleifen enthalten
  • Alles wird in Sequenzen angeordnet
  • Diese Konzepte funktionieren in JEDER Programmiersprache

🔌 Arduino-Programmierung

💡 Arduino - Hardware-nahe Programmierung

Arduino ist eine Plattform für Mikrocontroller-Programmierung. Hier siehst du, wie die gleichen Konzepte in der Hardware-Programmierung angewendet werden.

📟 Grundstruktur eines Arduino-Programms

🔍 Arduino Setup und Loop

// Arduino-Programm Grundstruktur void setup() { // Wird einmal beim Start ausgeführt Serial.begin(9600); // Serielle Kommunikation starten pinMode(LED_BUILTIN, OUTPUT); // Pin als Ausgang definieren } void loop() { // Wird endlos wiederholt digitalWrite(LED_BUILTIN, HIGH); // LED einschalten delay(1000); // 1 Sekunde warten digitalWrite(LED_BUILTIN, LOW); // LED ausschalten delay(1000); // 1 Sekunde warten }

🔄 Verzweigungen in Arduino

🔍 Arduino Sensor-Auswertung

int sensorPin = A0; // Analoger Pin A0 int ledPin = 13; // Digital Pin 13 int sensorValue = 0; // Variable für Sensorwert void setup() { Serial.begin(9600); pinMode(ledPin, OUTPUT); } void loop() { sensorValue = analogRead(sensorPin); // Sensor auslesen if (sensorValue > 500) { // Wenn Wert größer als 500 digitalWrite(ledPin, HIGH); // LED einschalten Serial.println("Sensor: HELL"); } else { // Sonst digitalWrite(ledPin, LOW); // LED ausschalten Serial.println("Sensor: DUNKEL"); } delay(100); // Kurz warten }

🔢 Schleifen in Arduino

🔍 Arduino for-Schleife

// LED mehrmals blinken lassen void blinkPattern() { for (int i = 0; i < 5; i++) { // 5 mal wiederholen digitalWrite(LED_BUILTIN, HIGH); // LED an delay(200); // 200ms warten digitalWrite(LED_BUILTIN, LOW); // LED aus delay(200); // 200ms warten } delay(1000); // Pause zwischen Mustern } void loop() { blinkPattern(); // Blinkmuster ausführen }

📝 Syntax-Grundlagen

⚠️ Wichtig zu verstehen

Programmiersprachen haben sehr strenge Regeln! Ein Computer kann nicht "raten" was gemeint ist - alles muss exakt richtig geschrieben werden.

🔤 Grundelemente der Syntax

Element Zweck Beispiel in Java Erklärung
Schlüsselwörter Reservierte Wörter der Sprache class, int, if, while Haben feste Bedeutung, können nicht als Namen verwendet werden
Runde Klammern () Methodenaufrufe & Parameter System.out.println("Text") Zeigen an, dass eine Methode aufgerufen wird
Geschweifte Klammern {} Code-Blöcke if (x > 5) { ... } Fassen zusammengehörigen Code zusammen
Semikolon ; Ende einer Anweisung int x = 5; Trennt Befehle voneinander
Kommentare Erklärungen für Menschen // Das ist ein Kommentar Werden vom Computer ignoriert

💬 Kommentare - Wichtige Notizen im Code

🔍 Arten von Kommentaren

// Zeilenkommentar - gilt nur für diese Zeile int alter = 18; // Kann auch am Ende einer Zeile stehen /* Blockkommentar - kann sich über mehrere Zeilen erstrecken Sehr nützlich für längere Erklärungen */ double pi = 3.14159; /** * JavaDoc-Kommentar * Spezielle Dokumentation für Methoden und Klassen * @param parameter Beschreibung des Parameters * @return Beschreibung des Rückgabewerts */

🎯 Warum sind Kommentare wichtig?

  • Erklären, WAS der Code macht
  • Erklären, WARUM bestimmte Entscheidungen getroffen wurden
  • Helfen anderen Programmierern (und dir selbst später!)
  • Werden vom Computer komplett ignoriert
  • Verlangsamen das Programm NICHT

🐛 Arten von Programmierfehlern

💡 Warum ist das wichtig?

Jeder Programmierer macht Fehler! Das Wichtige ist zu verstehen, welche Arten von Fehlern es gibt und wann sie auftreten.

1️⃣ Syntaxfehler (Grammatikfehler)

📖 Was sind Syntaxfehler?

Syntaxfehler sind wie Rechtschreibfehler in natürlichen Sprachen. Der Computer kann den Code nicht verstehen, weil die Grammatik der Programmiersprache nicht befolgt wurde.

🔍 Beispiele für Syntaxfehler in Java

// ❌ Falsch - fehlendes Semikolon int zahl = 5 // ✅ Richtig int zahl = 5; // ❌ Falsch - falsche Klammern if (x > 5) [ System.out.println("Groß"); } // ✅ Richtig if (x > 5) { System.out.println("Groß"); }

2️⃣ Laufzeitfehler

📖 Was sind Laufzeitfehler?

Laufzeitfehler treten auf, während das Programm ausgeführt wird. Der Code ist syntaktisch richtig, aber bei der Ausführung passiert etwas Unerwartetes.

🔍 Beispiele für Laufzeitfehler

// ❌ Division durch Null int a = 10; int b = 0; int ergebnis = a / b; // Programmabsturz! // ❌ Array-Index außerhalb der Grenzen int[] zahlen = {1, 2, 3}; int wert = zahlen[5]; // Index 5 existiert nicht! // ✅ Besser: Prüfung einbauen if (b != 0) { int ergebnis = a / b; }

3️⃣ Logische Fehler

📖 Was sind logische Fehler?

Logische Fehler sind die schwierigsten zu finden. Das Programm läuft ohne Absturz, macht aber nicht das, was der Programmierer wollte.

🔍 Beispiel für einen logischen Fehler

// Ziel: Durchschnitt von zwei Zahlen berechnen int a = 10; int b = 20; int durchschnitt = a + b / 2; // ❌ Falsch! // Ergebnis: 20 (nicht 15 wie gewünscht) // Grund: Division wird vor Addition ausgeführt // ✅ Richtig: int durchschnitt = (a + b) / 2; // Ergebnis: 15

📚 Zusammenfassung und Test

🎯 Das Wichtigste auf einen Blick

  • Es gibt viele Programmiersprachen, jede für bestimmte Zwecke optimiert
  • Computer verstehen nur Maschinensprache (0 und 1)
  • Höhere Sprachen werden durch Compiler oder Interpreter übersetzt
  • Es gibt drei Arten von Fehlern: Syntax-, Laufzeit- und logische Fehler
  • Alle Programme bestehen aus nur 3 Grundbausteinen: Sequenz, Auswahl, Wiederholung
  • Arduino zeigt Hardware-nahe Programmierung mit denselben Konzepten
  • Kommentare sind wichtig für Verständlichkeit und Dokumentation
