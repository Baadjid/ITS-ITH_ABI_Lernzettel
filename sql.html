🗄️ SQL - Structured Query Language

Die standardisierte Datenbanksprache für relationale Datenbanksysteme

📚 Was ist SQL?

🎯 SQL einfach erklärt

SQL (Structured Query Language) ist eine standardisierte Sprache zur Verwaltung relationaler Datenbanken. Stellen Sie sich vor, Sie haben einen riesigen Aktenschrank mit unzähligen Ordnern voller Informationen - SQL ist wie ein sehr intelligenter Assistent, der genau weiß, wo alles steht und Ihnen bei jeder Anfrage sofort die richtigen Informationen liefern kann.

Warum ist SQL wichtig?

  • ✅ Universell: Funktioniert mit fast allen Datenbanksystemen (MySQL, PostgreSQL, SQL Server, Oracle, etc.)
  • ✅ Mächtig: Von einfachen Abfragen bis zu komplexen Analysen
  • ✅ Standardisiert: Einmal gelernt, überall anwendbar
  • ✅ Praxisrelevant: In nahezu jedem IT-Bereich unverzichtbar

📋 Die vier Hauptbereiche von SQL

🔍 DQL - Data Query Language

Daten aus der Datenbank abfragen und anzeigen

SELECT

🏗️ DDL - Data Definition Language

Datenbankstruktur definieren und ändern

CREATE ALTER DROP

✏️ DML - Data Manipulation Language

Daten bearbeiten, hinzufügen und löschen

INSERT UPDATE DELETE

🔐 DCL - Data Control Language

Transaktionen und Zugriffsrechte kontrollieren

COMMIT ROLLBACK GRANT

🔍 SELECT - Die wichtigste SQL-Anweisung

📏 Aufbau einer SELECT-Anweisung

SELECT - Spalten auswählen

Bestimmt, welche Spalten angezeigt werden sollen

FROM - Tabelle angeben

Gibt an, aus welcher Tabelle die Daten kommen

WHERE - Bedingungen filtern

Filtert Zeilen nach bestimmten Kriterien (optional)

GROUP BY - Daten gruppieren

Fasst Zeilen nach gleichen Werten zusammen (optional)

HAVING - Gruppen filtern

Filtert die erstellten Gruppen (optional)

ORDER BY - Sortierung

Sortiert das Ergebnis aufsteigend oder absteigend (optional)

💡 Wichtige Reihenfolge merken!

Abarbeitungsreihenfolge (wie SQL intern arbeitet):

1. FROM → 2. WHERE → 3. GROUP BY → 4. HAVING → 5. SELECT → 6. ORDER BY

Diese Reihenfolge ist wichtig zu verstehen, auch wenn Sie anders schreiben!

🚀 Praktische SELECT-Beispiele

📝 Beispiel 1: Einfache Abfrage

Aufgabe: Zeige alle Spalten der Tabelle "Student" an

SELECT *
FROM Student;

Erklärung: Das * bedeutet "alle Spalten"

📝 Beispiel 2: Bestimmte Spalten auswählen

Aufgabe: Zeige nur Name und Email aller Studenten

SELECT Name, Email
FROM Student;

📝 Beispiel 3: Mit Bedingung (WHERE)

Aufgabe: Zeige alle Studenten im 3. Semester

SELECT Name, Email, Semester
FROM Student
WHERE Semester = 3;

📝 Beispiel 4: Sortierung

Aufgabe: Alle Studenten alphabetisch nach Namen sortiert

SELECT Name, Email
FROM Student
ORDER BY Name ASC;

ASC = aufsteigend (A-Z), DESC = absteigend (Z-A)

📝 Beispiel 5: Duplikate entfernen

Aufgabe: Alle verschiedenen Semester anzeigen

SELECT DISTINCT Semester
FROM Student;

DISTINCT entfernt doppelte Werte aus dem Ergebnis

📝 Beispiel 6: Spalten umbenennen

Aufgabe: Studentenname als "Vollname" anzeigen

SELECT Name AS Vollname, Email AS E_Mail_Adresse
FROM Student;

🔧 SQL-Operatoren und Vergleiche

🔢 Vergleichsoperatoren

  • Gleich =
  • Ungleich <> oder !=
  • Kleiner als <
  • Größer als >
  • Kleiner oder gleich <=
  • Größer oder gleich >=

🎯 Spezielle Operatoren

  • Bereich (inkl. Grenzen) BETWEEN
  • Liste von Werten IN
  • Muster suchen LIKE
  • Leere Werte IS NULL
  • Nicht leere Werte IS NOT NULL

🔗 Logische Operatoren

  • Beide Bedingungen erfüllt AND
  • Eine Bedingung erfüllt OR
  • Bedingung nicht erfüllt NOT

🔤 LIKE-Platzhalter

  • Beliebig viele Zeichen %
  • Genau ein Zeichen _

🎯 Praktische Operator-Beispiele

📝 BETWEEN - Bereich abfragen

Aufgabe: Alle Studenten zwischen dem 2. und 4. Semester

SELECT Name, Semester
FROM Student
WHERE Semester BETWEEN 2 AND 4;

Entspricht: WHERE Semester >= 2 AND Semester <= 4

📝 IN - Liste von Werten

Aufgabe: Studenten aus bestimmten Semestern

SELECT Name, Semester
FROM Student
WHERE Semester IN (1, 3, 5);

Entspricht: WHERE Semester = 1 OR Semester = 3 OR Semester = 5

📝 LIKE - Muster suchen

-- Namen die mit "M" beginnen
SELECT Name
FROM Student
WHERE Name LIKE 'M%';

-- Namen die mit "er" enden
SELECT Name
FROM Student
WHERE Name LIKE '%er';

-- Namen mit genau 4 Buchstaben
SELECT Name
FROM Student
WHERE Name LIKE '____';

📝 Komplexe Bedingungen mit AND/OR

Aufgabe: Studenten im 1. oder 2. Semester mit Gmail-Adresse

SELECT Name, Email, Semester
FROM Student
WHERE (Semester = 1 OR Semester = 2)
  AND Email LIKE '%@gmail.com';

⚠️ Wichtige Hinweise zu Operatoren

  • NULL-Werte: Verwenden Sie IS NULL, nicht = NULL
  • Klammern: Bei komplexen Bedingungen immer Klammern setzen!
  • Anführungszeichen: Texte in einfache Anführungszeichen: 'Text'
  • Groß-/Kleinschreibung: Bei LIKE oft relevant - abhängig von Datenbanksystem

📊 Aggregatfunktionen - Daten zusammenfassen

🎯 Was sind Aggregatfunktionen?

Aggregatfunktionen fassen mehrere Zeilen zu einem einzigen Wert zusammen. Stellen Sie sich vor, Sie haben eine Liste mit 1000 Verkäufen und wollen wissen: "Wie hoch war der Gesamtumsatz?" - genau dafür sind Aggregatfunktionen da!

🔢
COUNT()

Zählt die Anzahl der Datensätze

➕
SUM()

Addiert alle Werte einer Spalte

📈
AVG()

Berechnet den Durchschnitt

🏔️
MAX()

Findet den größten Wert

🏗️
MIN()

Findet den kleinsten Wert

📋 Beispieltabelle "Artikel"

ArtikelID Name Kategorie Preis Lagerbestand
1 Laptop Pro Elektronik 1299.99 15
2 Smartphone Elektronik 699.99 25
3 Schreibtisch Möbel 199.99 8
4 Stuhl Möbel 89.99 12

🎯 Aggregatfunktionen in der Praxis

📝 COUNT - Anzahl bestimmen

-- Wie viele Artikel gibt es insgesamt?
SELECT COUNT(*) AS Anzahl_Artikel
FROM Artikel;

-- Ergebnis: 4

-- Wie viele verschiedene Kategorien gibt es?
SELECT COUNT(DISTINCT Kategorie) AS Anzahl_Kategorien
FROM Artikel;

-- Ergebnis: 2

📝 SUM - Summe berechnen

-- Gesamtwert aller Artikel im Lager
SELECT SUM(Preis * Lagerbestand) AS Gesamtwert
FROM Artikel;

-- Gesamter Lagerbestand
SELECT SUM(Lagerbestand) AS Gesamt_Lagerbestand
FROM Artikel;

📝 AVG, MIN, MAX - Durchschnitt und Extreme

-- Durchschnittspreis aller Artikel
SELECT AVG(Preis) AS Durchschnittspreis
FROM Artikel;

-- Teuerstes und billigstes Produkt
SELECT 
    MAX(Preis) AS Hoechster_Preis,
    MIN(Preis) AS Niedrigster_Preis
FROM Artikel;

📊 GROUP BY - Gruppierung

GROUP BY ist sehr wichtig bei Aggregatfunktionen! Es teilt die Daten in Gruppen auf und wendet die Aggregatfunktion auf jede Gruppe einzeln an.

📝 Gruppierung nach Kategorie

-- Anzahl Artikel pro Kategorie
SELECT Kategorie, COUNT(*) AS Anzahl
FROM Artikel
GROUP BY Kategorie;

-- Ergebnis:
-- Elektronik: 2
-- Möbel: 2

-- Durchschnittspreis pro Kategorie
SELECT Kategorie, AVG(Preis) AS Durchschnitt
FROM Artikel
GROUP BY Kategorie;

📝 HAVING - Gruppen filtern

HAVING ist wie WHERE, aber für Gruppen nach GROUP BY!

-- Nur Kategorien mit mehr als 1 Artikel
SELECT Kategorie, COUNT(*) AS Anzahl
FROM Artikel
GROUP BY Kategorie
HAVING COUNT(*) > 1;

-- Kategorien mit Durchschnittspreis über 500€
SELECT Kategorie, AVG(Preis) AS Durchschnitt
FROM Artikel
GROUP BY Kategorie
HAVING AVG(Preis) > 500;

💡 WHERE vs. HAVING - Der wichtige Unterschied!

  • WHERE: Filtert einzelne Zeilen vor der Gruppierung
  • HAVING: Filtert Gruppen nach der Gruppierung
  • WHERE: Kann keine Aggregatfunktionen verwenden
  • HAVING: Wird meist mit Aggregatfunktionen verwendet

⚠️ Häufige Fehler bei Aggregatfunktionen

  • Fehler: Normale Spalten ohne GROUP BY bei Aggregatfunktionen
  • Richtig: Alle nicht-aggregierten Spalten müssen in GROUP BY stehen
  • Fehler: COUNT(Spaltenname) zählt nur nicht-NULL Werte
  • Richtig: COUNT(*) zählt alle Zeilen inklusive NULL

🔗 JOIN - Tabellen verknüpfen

🎯 Was ist ein JOIN?

Ein JOIN verknüpft Daten aus zwei oder mehr Tabellen basierend auf gemeinsamen Werten. Stellen Sie sich vor, Sie haben zwei Adressbücher - eines mit Namen und Telefonnummern, ein anderes mit Namen und E-Mail-Adressen. Ein JOIN würde diese zusammenfügen, sodass Sie alle Informationen zu einer Person an einem Ort haben.

📋 Beispieltabellen für JOINs

Tabelle "Student"

StudentID Name Email
1 Max Müller max@uni.de
2 Anna Schmidt anna@uni.de
3 Tom Weber tom@uni.de

Tabelle "Student_Kurs"

StudentID KursID Note
1 101 2.0
1 102 1.5
2 101 1.8

Tabelle "Kurs"

KursID Kursname Credits Dozent
101 Mathematik I 6 Prof. Euler
102 Informatik I 8 Prof. Turing
103 Physik I 6 Prof. Newton

🔗 JOIN-Arten im Überblick

INNER
INNER JOIN

Nur Datensätze, die in beiden Tabellen existieren

LEFT
LEFT JOIN

Alle Datensätze der linken Tabelle + passende der rechten

RIGHT
RIGHT JOIN

Alle Datensätze der rechten Tabelle + passende der linken

FULL
FULL OUTER

Alle Datensätze beider Tabellen

🎯 INNER JOIN - Die wichtigste JOIN-Art

📝 Einfacher INNER JOIN

Aufgabe: Zeige alle Studenten mit ihren Kursen und Noten

SELECT s.Name, sk.Note
FROM Student s
INNER JOIN Student_Kurs sk ON s.StudentID = sk.StudentID;

Erklärung:

  • s und sk sind Tabellen-Aliase (Abkürzungen)
  • ON definiert die JOIN-Bedingung (wie die Tabellen verknüpft werden)
  • Nur Studenten, die auch Kurse haben, werden angezeigt

📝 JOIN mit drei Tabellen

Aufgabe: Zeige Studentenname, Kursname und Note

SELECT s.Name, k.Kursname, sk.Note
FROM Student s
INNER JOIN Student_Kurs sk ON s.StudentID = sk.StudentID
INNER JOIN Kurs k ON sk.KursID = k.KursID;

Faustregel: Für n Tabellen benötigen Sie mindestens (n-1) JOIN-Bedingungen!

📝 Alte WHERE-Syntax (Equi-Join)

Früher wurde oft die WHERE-Syntax verwendet (funktioniert noch, ist aber nicht mehr empfohlen):

-- Alte Schreibweise (nicht empfohlen)
SELECT s.Name, sk.Note
FROM Student s, Student_Kurs sk
WHERE s.StudentID = sk.StudentID;

-- Moderne Schreibweise (empfohlen)
SELECT s.Name, sk.Note
FROM Student s
INNER JOIN Student_Kurs sk ON s.StudentID = sk.StudentID;

🔄 Andere JOIN-Arten

📝 LEFT JOIN - Alle linken Datensätze

Aufgabe: Alle Studenten anzeigen, auch die ohne Kurse

SELECT s.Name, sk.Note
FROM Student s
LEFT JOIN Student_Kurs sk ON s.StudentID = sk.StudentID;

Ergebnis: Alle Studenten werden angezeigt. Studenten ohne Kurse haben NULL bei der Note.

💡 JOIN-Tipps für die Praxis

  • Aliase verwenden: Macht SQL lesbarer: FROM Student s
  • INNER JOIN: Meist das, was Sie brauchen (90% der Fälle)
  • LEFT JOIN: Wenn Sie "alle aus Tabelle A" wollen
  • JOIN-Bedingung prüfen: Meist Primärschlüssel = Fremdschlüssel
  • Performance: JOINs sind meist schneller als mehrere separate Abfragen

⚠️ Häufige JOIN-Fehler

  • Vergessene JOIN-Bedingung: Führt zu Kreuzprodukt (sehr viele Ergebnisse!)
  • Falsche JOIN-Art: INNER statt LEFT JOIN kann Daten "verschwinden" lassen
  • Mehrdeutige Spaltennamen: Ohne Alias unclear bei gleichnamigen Spalten

✏️ DML - Daten manipulieren (INSERT, UPDATE, DELETE)

🎯 Data Manipulation Language (DML)

Mit DML-Befehlen verändern Sie den Inhalt der Datenbank - im Gegensatz zu DDL, was die Struktur ändert. Die drei wichtigsten Befehle sind INSERT (hinzufügen), UPDATE (ändern) und DELETE (löschen).

➕ INSERT - Neue Daten hinzufügen

📝 Einzelnen Datensatz einfügen

-- Neuen Student hinzufügen
INSERT INTO Student (StudentID, Name, Email, Semester)
VALUES (4, 'Lisa Meyer', 'lisa@uni.de', 2);

-- Wenn alle Spalten gefüllt werden, kann man Spaltennamen weglassen
INSERT INTO Student
VALUES (5, 'Tim Koch', 'tim@uni.de', 1);

📝 Mehrere Datensätze auf einmal

-- Mehrere Studenten gleichzeitig hinzufügen
INSERT INTO Student (StudentID, Name, Email, Semester)
VALUES 
    (6, 'Sarah Fischer', 'sarah@uni.de', 3),
    (7, 'Jan Hoffmann', 'jan@uni.de', 1),
    (8, 'Nina Braun', 'nina@uni.de', 4);

📝 Daten aus anderer Tabelle kopieren

-- Backup aller Studenten im höheren Semester erstellen
INSERT INTO StudentBackup
SELECT * 
FROM Student 
WHERE Semester > 5;

🔄 UPDATE - Bestehende Daten ändern

📝 Einzelnen Datensatz aktualisieren

-- Email-Adresse eines bestimmten Studenten ändern
UPDATE Student
SET Email = 'max.mueller@uni.de'
WHERE StudentID = 1;

📝 Mehrere Spalten gleichzeitig ändern

-- Name und Email eines Studenten aktualisieren
UPDATE Student
SET 
    Name = 'Maximilian Müller',
    Email = 'maximilian.mueller@uni.de'
WHERE StudentID = 1;

📝 Mehrere Datensätze auf einmal ändern

-- Preiserhöhung um 5% für alle Elektronik-Artikel
UPDATE Artikel
SET Preis = Preis * 1.05
WHERE Kategorie = 'Elektronik';

-- Alle Erstsemester ins 2. Semester versetzen
UPDATE Student
SET Semester = Semester + 1
WHERE Semester = 1;

🗑️ DELETE - Daten löschen

📝 Bestimmte Datensätze löschen

-- Einen bestimmten Studenten löschen
DELETE FROM Student
WHERE StudentID = 8;

-- Alle Studenten über dem 10. Semester löschen
DELETE FROM Student
WHERE Semester > 10;

📝 Alle Datensätze löschen

-- Alle Daten aus der Backup-Tabelle löschen (Tabelle bleibt bestehen)
DELETE FROM StudentBackup;

-- Alternativ: TRUNCATE (schneller, aber weniger flexibel)
TRUNCATE TABLE StudentBackup;

⚠️ ACHTUNG: Sicherheit bei DML-Befehlen!

  • WHERE-Klausel nicht vergessen! DELETE FROM Student löscht ALLE Studenten!
  • Erst testen: Bei UPDATE/DELETE erst mit SELECT prüfen, welche Datensätze betroffen sind
  • Backup machen: Vor größeren Änderungen immer ein Backup erstellen
  • Transaktionen nutzen: Bei kritischen Operationen (siehe nächster Abschnitt)

💡 Profi-Tipps für DML

  • Test-SELECT: SELECT * FROM Student WHERE StudentID = 1 vor UPDATE/DELETE
  • Betroffene Zeilen prüfen: Die meisten Tools zeigen an, wie viele Zeilen geändert wurden
  • NULL-Werte beachten: SET Spalte = NULL löscht den Wert (falls erlaubt)
  • Berechnungen in UPDATE: SET Preis = Preis * 1.1 ist möglich

🏗️ DDL - Datenbankstruktur definieren (CREATE, ALTER, DROP)

🎯 Data Definition Language (DDL)

DDL-Befehle verändern die Struktur der Datenbank - sie erstellen, ändern oder löschen Tabellen, Spalten und andere Datenbankobjekte. Denken Sie daran wie an das Bauen eines Hauses: Erst die Struktur (DDL), dann die Einrichtung (DML).

🏠 CREATE TABLE - Neue Tabelle erstellen

📝 Einfache Tabelle erstellen

CREATE TABLE Student (
    StudentID NUMBER(4) PRIMARY KEY,
    Name VARCHAR(50) NOT NULL,
    Email VARCHAR(100) UNIQUE,
    Semester NUMBER(2),
    Geburtsdatum DATE
);

Erklärung der Constraints (Beschränkungen):

  • PRIMARY KEY: Eindeutiger Hauptschlüssel (automatisch NOT NULL und UNIQUE)
  • NOT NULL: Feld muss ausgefüllt werden
  • UNIQUE: Wert darf nur einmal vorkommen
  • VARCHAR(n): Text mit maximal n Zeichen
  • NUMBER(n): Zahl mit maximal n Stellen

📝 Tabelle mit Fremdschlüssel

CREATE TABLE Student_Kurs (
    StudentID NUMBER(4),
    KursID NUMBER(4),
    Note DECIMAL(2,1),
    Anmeldedatum DATE DEFAULT SYSDATE,
    
    -- Primärschlüssel aus beiden Spalten
    PRIMARY KEY (StudentID, KursID),
    
    -- Fremdschlüssel-Beziehungen
    FOREIGN KEY (StudentID) REFERENCES Student(StudentID),
    FOREIGN KEY (KursID) REFERENCES Kurs(KursID)
);

🔧 ALTER TABLE - Tabelle ändern

📝 Spalte hinzufügen

-- Neue Spalte hinzufügen
ALTER TABLE Student 
ADD Telefon VARCHAR(20);

-- Mehrere Spalten gleichzeitig
ALTER TABLE Student 
ADD (
    Adresse VARCHAR(100),
    PLZ VARCHAR(5)
);

📝 Spalte ändern

-- Spaltengröße ändern
ALTER TABLE Student 
MODIFY Name VARCHAR(100);

-- Spalte zu NOT NULL machen
ALTER TABLE Student 
MODIFY Email VARCHAR(100) NOT NULL;

📝 Spalte löschen

-- Einzelne Spalte löschen
ALTER TABLE Student 
DROP COLUMN Telefon;

-- Mehrere Spalten löschen
ALTER TABLE Student 
DROP (Adresse, PLZ);

🗑️ DROP TABLE - Tabelle löschen

📝 Tabelle komplett löschen

-- Tabelle mit allen Daten löschen
DROP TABLE StudentBackup;

-- Sicher löschen (nur wenn Tabelle existiert)
DROP TABLE IF EXISTS StudentBackup;

⚠️ ACHTUNG: DDL ist meist nicht rückgängig zu machen!

  • DROP TABLE: Löscht die Tabelle UND alle Daten unwiderruflich!
  • DROP COLUMN: Alle Daten in dieser Spalte sind weg!
  • ALTER TABLE: Strukturänderungen können Datentyp-Konflikte verursachen
  • Backup erstellen: Vor großen Strukturänderungen immer!

💡 DDL Best Practices

  • Naming Conventions: Einheitliche Namen für Tabellen und Spalten
  • Constraints definieren: PRIMARY KEY, FOREIGN KEY etc. von Anfang an
  • Datentypen durchdenken: VARCHAR-Größe großzügig, aber nicht übertrieben
  • Testumgebung nutzen: Strukturänderungen erst testen, dann produktiv

🔐 DCL - Transaktionen (COMMIT, ROLLBACK)

🎯 Was sind Transaktionen?

Eine Transaktion ist eine logische Arbeitseinheit, die entweder vollständig ausgeführt oder vollständig rückgängig gemacht wird. Stellen Sie sich vor, Sie überweisen Geld: Das Geld muss gleichzeitig von einem Konto abgebucht UND auf dem anderen gutgeschrieben werden - niemals nur eins davon!

📝 Das klassische Banküberweisungsbeispiel

-- Geldüberweisung als Transaktion
START TRANSACTION;

-- Schritt 1: Geld vom ersten Konto abbuchen
UPDATE Konto 
SET Betrag = Betrag - 100 
WHERE KontoNr = 1;

-- Schritt 2: Geld auf zweites Konto einzahlen
UPDATE Konto 
SET Betrag = Betrag + 100 
WHERE KontoNr = 2;

-- Wenn alles OK: Änderungen bestätigen
COMMIT;

-- Bei Fehler: Alles rückgängig machen
-- ROLLBACK;

Wichtig: Entweder werden BEIDE Updates ausgeführt, oder KEINES - niemals nur eins!

🔧 Die wichtigsten Transaktions-Befehle

✅ COMMIT

Bestätigt alle Änderungen und macht sie dauerhaft

COMMIT;

↩️ ROLLBACK

Macht alle Änderungen seit dem letzten COMMIT rückgängig

ROLLBACK;

🔖 SAVEPOINT

Setzt einen Zwischenpunkt für teilweises Rollback

SAVEPOINT punkt1;
ROLLBACK TO punkt1;

📝 Praktisches Beispiel mit SAVEPOINT

START TRANSACTION;

-- Erste Gruppe von Änderungen
INSERT INTO Student VALUES (10, 'Test User', 'test@uni.de', 1);

-- Zwischenpunkt setzen
SAVEPOINT nach_insert;

-- Zweite Gruppe von Änderungen
UPDATE Student SET Semester = 2 WHERE StudentID = 10;

-- Ups, Fehler gemacht! Nur UPDATE rückgängig machen
ROLLBACK TO nach_insert;

-- Student ist noch da, aber Semester-Update ist weg
COMMIT;

⚗️ ACID-Prinzip

Transaktionen folgen dem ACID-Prinzip - vier wichtige Eigenschaften:

🔒 Atomicity

Atomarität: Alles oder nichts - eine Transaktion wird entweder vollständig oder gar nicht ausgeführt

✅ Consistency

Konsistenz: Die Datenbank bleibt in einem gültigen Zustand (alle Regeln werden befolgt)

🏝️ Isolation

Isolation: Parallele Transaktionen beeinflussen sich nicht gegenseitig

💾 Durability

Dauerhaftigkeit: Bestätigte Änderungen bleiben auch bei Systemausfall erhalten

💡 Wann Transaktionen verwenden?

  • Zusammengehörige Änderungen: Überweisungen, Bestellungen, etc.
  • Kritische Operationen: Wenn Datenkonsistenz wichtig ist
  • Mehrere DML-Befehle: Die logisch zusammengehören
  • Fehlerbehandlung: Möglichkeit zum Rückgängigmachen

⚠️ Wichtige Hinweise zu Transaktionen

  • Autocommit: Viele Tools machen automatisch COMMIT - prüfen Sie die Einstellungen!
  • Lange Transaktionen: Blockieren andere Benutzer - kurz halten!
  • DDL-Befehle: Machen oft automatisch COMMIT (CREATE, DROP, ALTER)
  • Verbindungsabbruch: Macht automatisch ROLLBACK

🎯 Abitur-relevante Tipps & häufige Aufgaben

📋 Typische Abitur-Aufgaben zu SQL

Im Abitur kommen SQL-Aufgaben meist in Verbindung mit Datenbankentwurf vor. Sie bekommen oft eine Textbeschreibung und sollen dann ERM erstellen, Tabellen ableiten und SQL-Abfragen schreiben.

🏆 Top 10 Aufgabentypen

Aufgabe 1: Einfache SELECT mit WHERE

Typische Frage: "Geben Sie alle Schüler der Klasse 12a aus"

SELECT *
FROM Schüler
WHERE Klasse = '12a';

Aufgabe 2: Aggregatfunktionen mit GROUP BY

Typische Frage: "Wie viele Schüler sind in jeder Klasse?"

SELECT Klasse, COUNT(*) AS Anzahl_Schüler
FROM Schüler
GROUP BY Klasse;

Aufgabe 3: JOIN zwischen zwei Tabellen

Typische Frage: "Zeigen Sie alle Schüler mit ihren Noten"

SELECT s.Name, n.Fach, n.Note
FROM Schüler s
INNER JOIN Noten n ON s.SchülerID = n.SchülerID;

Aufgabe 4: Durchschnitt mit HAVING

Typische Frage: "Welche Schüler haben einen Notendurchschnitt besser als 2.0?"

SELECT s.Name, AVG(n.Note) AS Durchschnitt
FROM Schüler s
INNER JOIN Noten n ON s.SchülerID = n.SchülerID
GROUP BY s.SchülerID, s.Name
HAVING AVG(n.Note) < 2.0;

Aufgabe 5: INSERT mit mehreren Datensätzen

Typische Frage: "Fügen Sie drei neue Schüler hinzu"

INSERT INTO Schüler (Name, Klasse, Geburtsdatum)
VALUES
    ('Max Mustermann', '12a', '2006-05-15'),
    ('Anna Schmidt', '12b', '2006-03-22'),
    ('Tom Weber', '12a', '2006-08-10');

Aufgabe 6: UPDATE mit Bedingung

Typische Frage: "Alle Schüler der Klasse 12a kommen in die Klasse 13a"

UPDATE Schüler
SET Klasse = '13a'
WHERE Klasse = '12a';

Aufgabe 7: DELETE mit komplexer Bedingung

Typische Frage: "Löschen Sie alle Noten schlechter als 4.0 im Fach Mathematik"

DELETE FROM Noten
WHERE Fach = 'Mathematik'
  AND Note > 4.0;

Aufgabe 8: LIKE mit Platzhaltern

Typische Frage: "Alle Schüler deren Name mit 'M' beginnt und auf 'er' endet"

SELECT *
FROM Schüler
WHERE Name LIKE 'M%er';

Aufgabe 9: Sortierung mit mehreren Kriterien

Typische Frage: "Sortieren Sie nach Klasse aufsteigend, dann nach Name absteigend"

SELECT *
FROM Schüler
ORDER BY Klasse ASC, Name DESC;

Aufgabe 10: Komplexer JOIN mit drei Tabellen

Typische Frage: "Zeigen Sie Schülername, Lehrername und Fach für alle Noten"

SELECT s.Name AS Schüler, l.Name AS Lehrer, f.Fachname, n.Note
FROM Schüler s
INNER JOIN Noten n ON s.SchülerID = n.SchülerID
INNER JOIN Fach f ON n.FachID = f.FachID
INNER JOIN Lehrer l ON f.LehrerID = l.LehrerID;

📝 Häufige Fehler und wie man sie vermeidet

❌ Die 10 häufigsten SQL-Fehler im Abitur

  1. Vergessene WHERE-Klausel: DELETE FROM Schüler löscht ALLE!
  2. Fehlende JOIN-Bedingung: Führt zu Kreuzprodukt (riesige Ergebnismenge)
  3. GROUP BY vergessen: Bei Aggregatfunktionen mit anderen Spalten
  4. Falsche Anführungszeichen: Texte in einfache: 'Text'
  5. NULL-Vergleiche: IS NULL statt = NULL
  6. HAVING statt WHERE: HAVING nur nach GROUP BY verwenden
  7. Klammern bei komplexen Bedingungen: (A AND B) OR C
  8. Aliase bei mehrdeutigen Spalten: s.Name, k.Name
  9. Falsche JOIN-Art: INNER vs. LEFT JOIN beachten
  10. Sortierung vergessen: ORDER BY für definierte Reihenfolge

🎯 Abitur-Strategie: Schritt-für-Schritt Vorgehen

📋 SQL-Aufgaben systematisch lösen

  1. Aufgabe genau lesen: Was wird gefragt? Welche Tabellen sind beteiligt?
  2. Benötigte Tabellen identifizieren: Welche Informationen brauche ich?
  3. JOIN-Bedingungen finden: Wie sind die Tabellen verknüpft?
  4. WHERE-Bedingungen sammeln: Welche Filter sind nötig?
  5. SELECT-Teil bestimmen: Welche Spalten sollen angezeigt werden?
  6. Aggregation prüfen: Brauche ich COUNT, AVG, etc.?
  7. Sortierung überlegen: ORDER BY nötig?
  8. Schritt für Schritt schreiben: Erst FROM, dann JOIN, dann WHERE...
  9. Testen (wenn möglich): Ergebnis plausibel?
  10. Doppelt prüfen: Alle Bedingungen erfüllt?

INNER: "Nur gemeinsame Freunde"

RIGHT: "Alle Freunde von rechts + gemeinsame"

COUNT: "Zähl mal!"

MIN/MAX: "Kleinstes/Größtes!"

💡 LIKE-Platzhalter

%: "Prozent für viele Zeichen"

_: "Unterstrich für ein Zeichen"

Beispiel: 'M%er' = "Müller", "Meier", "Mayer"

⏱️ Last-Minute Checkliste

📋 Vor der Klausur nochmal prüfen

  • ✅ SELECT-Grundstruktur: SELECT - FROM - WHERE - GROUP BY - HAVING - ORDER BY
  • ✅ JOIN-Syntax: FROM Tabelle1 INNER JOIN Tabelle2 ON Bedingung
  • ✅ Aggregatfunktionen: COUNT(*), SUM(), AVG(), MIN(), MAX()
  • ✅ DML-Befehle: INSERT INTO, UPDATE SET WHERE, DELETE FROM WHERE
  • ✅ Operatoren: =, <>, LIKE, BETWEEN, IN, IS NULL
  • ✅ Logik: AND, OR, NOT mit Klammern bei komplexen Bedingungen
  • ✅ Anführungszeichen: Texte in einfache Anführungszeichen: 'Text'
  • ✅ WHERE vs. HAVING: WHERE vor Gruppierung, HAVING nach Gruppierung
  • ✅ NULL-Werte: IS NULL / IS NOT NULL verwenden
  • ✅ Aliase: Tabellen- und Spalten-Aliase für Übersichtlichkeit
